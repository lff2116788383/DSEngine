# DSEngine 英文专业术语速查表

>
> 格式：**缩写** → 英文全称 → 中文名称 → 大白话解释 → 打个比方

## 一、渲染管线基础

| 缩写 | 英文全称 | 中文名称 | 大白话解释 | 打个比方 |
|:----:|---------|:--------:|-----------|---------|
| **PBR** | Physically Based Rendering | 基于物理的渲染 | 用物理公式模拟光线在真实世界中怎么反射，让金属/木头/皮肤看起来逼真 | 像用公式算「光照到物体上该是什么颜色」，不靠画师凭感觉调 |
| **HDR** | High Dynamic Range | 高动态范围 | 让画面最亮和最暗之间的层次更丰富，亮的够亮、暗的够暗但细节都不丢 | 像手机拍照开 HDR，天空不会一片白、阴影不会一片黑 |
| **SDR** | Standard Dynamic Range | 标准动态范围 | 传统的显示范围，最亮和最暗之间的层次有限，亮部容易过曝、暗部容易死黑 | 像老式 CRT 电视——亮的地方白成一片，暗的地方糊成一团，不像 HDR 那么细腻 |
| **BRDF** | Bidirectional Reflectance Distribution Function | 双向反射分布函数 | 描述光线打到表面上后往哪些方向反射的数学公式，决定材质是粗糙还是光滑 | 好比告诉你「光照在木头上会往哪弹」，决定它是哑光还是反光 |
| **IBL** | Image-Based Lighting | 基于图像的光照 | 用一张全景图（环境贴图）来照亮场景，而不是一个个手动摆灯光 | 像在户外拍照——天空和周围的环境自然地把光打到物体上，不用自己打灯 |
| **GGX** | GGX Distribution | GGX 法线分布函数 | 一种计算高光形状的数学公式，PBR 最常用的高光算法 | 决定了金属表面的高光点是一个小亮点还是大片的模糊反光 |
| **SH** | Spherical Harmonics | 球谐函数 | 用一组数学函数来压缩和近似复杂的灯光信息，常用于存储光照探针 | 像把一张全景图压缩成几个数字，用的时候再还原出大致的光照效果 |
| **ACES** | Academy Color Encoding System | 学院色彩编码系统 | 电影行业标准的色彩处理流程，比普通调色更真实自然 | 像手机拍照的「专业模式」vs「普通模式」，ACES 颜色更准、过渡更自然 |
| **LUT** | Look Up Table | 颜色查找表 | 一个颜色映射表，把输入的颜色批量转换成另一种风格的输出颜色 | 像手机拍照的「滤镜」——一整套风格颜色，套上去画面就变调了 |
| **sRGB** | Standard Red Green Blue | 标准 RGB 色彩空间 | 显示器和网页通用的颜色标准，大多数图片都使用这个色彩空间 | 像大家都说同一种「红色」的标准是什么，避免不同屏幕显示不一样 |
| **SDF** | Signed Distance Field | 有符号距离场 | 用数值记录每个点到物体表面的最近距离，正数在外面、负数在里面 | 像等高线地图——每条线上标着「离山顶多远」，用来高效渲染字体和形状 |
| **OIT** | Order Independent Transparency | 顺序无关透明度 | 不管半透明物体从哪个角度渲染，都能正确混合显示的技术 | 隔着有色玻璃看另一个玻璃杯，不管先画哪个，最终效果都是对的 |
| **GBuffer** | Geometry Buffer | 几何缓冲区 | 延迟渲染中用来存场景几何信息（位置/法线/颜色等）的一组纹理，一次渲染多次使用 | 像拍电影时分好几个图层——前景、背景、道具分别拍，最后再合成到一起 |
| **Deferred Rendering** | Deferred Rendering（原名） | 延迟渲染 | 先一口气把所有物体的几何信息存到 GBuffer，再统一做光照计算，避免无效的光照计算 | 像先拍完所有演员的绿幕镜头（存 GBuffer），再统一加背景特效（光照），省得每个镜头单独打光 |
| **Clustered Forward+** | Clustered Forward+（原名） | 分簇前向渲染 | 一种光源管理技术，把屏幕分成很多小格子，每个格子只计算影响它的那几盏灯，支持几百盏灯光 | 像演唱会每个区域只开该区域的灯——不用全场几千盏灯都算一遍，只算你周围的那几盏 |
| **Ray Tracing** | Ray Tracing（原名） | 光线追踪 | 模拟光线在场景中弹射的渲染技术，能实现最真实的反射/阴影/折射，但性能消耗巨大 | 像用激光笔从眼睛发射光线，追踪它怎么反弹、最终照到了什么——画面最真实，但也最费显卡 |

## 一.b、全局光照（进阶渲染）

| 缩写 | 英文全称 | 中文名称 | 大白话解释 | 打个比方 |
|:----:|---------|:--------:|-----------|---------|
| **VCT** | Voxel Cone Tracing | 体素锥追踪 | 一种全局光照技术，把场景转换成小方块（体素）来快速计算光线反弹效果 | 像用乐高搭出整个场景，然后从每个点向四周发射短光线，看周围乐高块的颜色再反弹回来 |
| **SVOGI** | Sparse Voxel Octree Global Illumination | 稀疏体素八叉树全局光照 | VCT 的优化版，只在有物体的地方建体素，空的地方跳过，节省计算量 | 像只给房间里摆家具的地方画地图，空地板不画——省事又够用 |
| **VSM** | Variance Shadow Maps | 方差阴影贴图 | 一种阴影抗锯齿技术，用统计学方法让阴影边缘平滑 | 像给阴影边缘加了模糊滤镜，看起来不刺眼 |
| **ESM** | Exponential Shadow Maps | 指数阴影贴图 | 另一种阴影抗锯齿技术，用指数函数来平滑阴影边缘 | 和 VSM 类似，但算法不同，各有优劣 |

## 二、后处理特效

| 缩写 | 英文全称 | 中文名称 | 大白话解释 | 打个比方 |
|:----:|---------|:--------:|-----------|---------|
| **Bloom** | Bloom（原名，意为"绽放"） | 泛光效果 | 让画面中特别亮的部分产生柔和发光，向周围晕开 | 阳光太强时看灯周围有光晕，Bloom 就是在游戏里模拟这种感觉 |
| **SSAO** | Screen Space Ambient Occlusion | 屏幕空间环境光遮蔽 | 在物体靠得很近的缝隙处自动加深阴影，增强立体感 | 像墙角、桌腿和地面接触的地方自动变暗，看起来更有重量感 |
| **TAA** | Temporal Anti-Aliasing | 时间抗锯齿 | 利用前后好几帧的图像信息来消除锯齿和闪烁 | 像把多张照片叠在一起平均，边缘的锯齿就被磨平了 |
| **FXAA** | Fast Approximate Anti-Aliasing | 快速近似抗锯齿 | 一种对性能影响很小的抗锯齿方法，直接在最终画面上模糊锯齿边 | 像在拍好的照片上直接 PS 模糊锯齿边缘，速度快但会有点糊 |
| **SSR** | Screen Space Reflection | 屏幕空间反射 | 利用屏幕已经画好的画面来模拟反射效果，水面、镜面都会反光 | 像镜子照出对面物体——SSR 用屏幕上已有的像素来算反射，省性能 |
| **DOF** | Depth of Field | 景深效果 | 让焦距外的物体变模糊，模拟真实摄像机的对焦效果 | 像拍照时对焦在前面的花上，背景就虚化了，突出主体 |
| **CSM** | Cascaded Shadow Maps | 级联阴影贴图 | 把阴影按距离分成好几层，近处用高清阴影，远处用低清阴影 | 像近处看高清照片、远处看缩略图——省性能又保证近处画质 |
| **PCSS** | Percentage Closer Soft Shadows | 百分比渐近软阴影 | 让阴影边缘根据光源大小和距离产生由硬到柔的过渡 | 晴天阴影边缘锐利，阴天阴影模糊——PCSS 模拟这种自然变化 |
| **Motion Blur** | Motion Blur（原名） | 运动模糊 | 让快速移动的物体产生拖影效果，增加动感 | 像拍高速赛车时照片会模糊，Motion Blur 模拟这种视觉残留 |
| **Vignette** | Vignette（原名） | 暗角效果 | 画面四角自然变暗，中间亮，模仿镜头的光学特性 | 像老旧照片四个角发暗，让视线自然聚焦到画面中央 |
| **Film Grain** | Film Grain（原名） | 胶片颗粒 | 在画面上叠加细微的随机噪点，模仿胶片电影的质感 | 像老电影画面上的细小颗粒，增加复古电影感 |
| **Color Grading** | Color Grading | 颜色调色 | 调整整幅画面的色调，营造特定的情绪氛围 | 像电影后期调色——冷色调给人压抑感，暖色调给人温暖感 |
| **Auto Exposure** | Auto Exposure / Eye Adaptation | 自动曝光（人眼适应） | 模拟人眼从亮处走到暗处时瞳孔自动调节的过程 | 从室外走进电影院，刚开始看不清，过几秒就适应了——游戏里也这样 |
| **Contact Shadow** | Contact Shadow（原名） | 接触阴影 | 在两个物体接触的地方增加精细的阴影，增强细节真实感 | 像杯子放在桌面上，杯底和桌面接触的地方有一圈细细的阴影 |
| **Color Banding** | Color Banding（原名） | 颜色断层 | 画面颜色过渡出现明显的阶梯状色块，是一种画质瑕疵 | 像渐变天空出现一圈圈的色环，而不是平滑过渡——一般是要消除的问题 |
| **Dither** | Dithering | 抖动/递色 | 通过随机微调颜色值来消除颜色断层，让颜色过渡更平滑 | 像用不同颜色的细点密集排列，远看就混成了一种渐变色 |

## 三、材质与着色

| 缩写 | 英文全称 | 中文名称 | 大白话解释 | 打个比方 |
|:----:|---------|:--------:|-----------|---------|
| **SSS** | Subsurface Scattering | 次表面散射 | 光线穿入半透明材质（如皮肤/树叶/蜡）内部再散射出来的效果 | 像手电筒照在手指上，边缘透出红光——光线穿过了皮肤内部 |
| **POM** | Parallax Occlusion Mapping | 视差遮挡贴图 | 在平面上模拟出凹凸不平的立体感，不用真的加 3D 模型 | 像一张砖墙照片看起来砖块是立体的，用手指摸其实还是平的 |
| **Toon Shading** | Toon Shading / Cel Shading | 卡通着色 | 用阶梯式的色块代替平滑颜色过渡，模仿动画/漫画的画风 | 像《原神》《无主之地》那种角色有明显的明暗分界线，不是真实照片风格 |
| **Cel Shading** | Cel Shading（同上） | 赛璐珞着色 | Toon Shading 的另一种说法，来源于赛璐珞动画片制作工艺 | 同上，日本动漫那种色块分明的画风 |
| **Clear Coat** | Clear Coat（原名） | 清漆层 | 在基础颜色上叠加一层透明光亮层，模拟车漆或木器清漆的效果 | 像给木桌上了一层清漆，既能看见木纹，表面又有光滑反光 |
| **Anisotropy** | Anisotropic Reflection | 各向异性反射 | 金属拉丝或头发丝那种顺着一个方向拉伸的高光反射效果 | 像不锈钢水槽上的拉丝纹理——高光是顺着一个方向拉长的，不是圆形 |
| **Half-Lambert** | Half Lambert | 半兰伯特光照 | 一种让暗面不会完全变黑的漫反射算法，常用于游戏角色 | 像给角色背光面补了一点光，不会出现死黑一片，看着更柔和 |
| **NPR** | Non-Photorealistic Rendering | 非真实感渲染 | 不追求像照片一样真实，追求特定艺术风格的渲染方式 | 水彩画风、动漫风、素描风都属于 NPR——不真实但好看 |
| **DSSL** | DSE Shading Language | DSE 着色器语言 | 引擎自己定义的一种材质描述语言，写起来比 GLSL/HLSL 简单 | 像写配方而不是写代码——告诉引擎「我想要什么材质」，引擎自动翻译给显卡 |

## 四、图形 API（底层接口）

| 缩写 | 英文全称 | 中文名称 | 大白话解释 | 打个比方 |
|:----:|---------|:--------:|-----------|---------|
| **RHI** | Rendering Hardware Interface | 渲染硬件接口 | 引擎抽象出来的一层，让同一份代码能跑在 OpenGL/Vulkan/D3D11 上 | 像充电器转换插头——不管美标还是欧标插座，换个头就能用 |
| **OpenGL** | Open Graphics Library | 开放图形库 | 最广泛使用的跨平台图形 API，老但兼容性强，几乎任何显卡都支持 | 像通用钥匙——大部分门都能开，但不是最新的技术 |
| **Vulkan** | Vulkan（原名） | Vulkan 图形 API | 新一代高性能图形 API，CPU 开销小，适合复杂场景 | 像专业工具箱——功能强大但需要自己拼装，适合高手 |
| **D3D11** | Direct3D 11 | 微软 Direct3D 11 | Windows 系统上的主流图形 API，Win7 及以上都支持 | 像 Windows 的亲儿子——Windows 电脑上兼容性最好 |
| **D3D12** | Direct3D 12 | 微软 Direct3D 12 | 新一代 Windows 图形 API，类似 Vulkan，CPU 开销更低 | 像 Windows 亲儿子的升级版——更强但只支持 Win10+ |
| **GLSL** | OpenGL Shading Language | OpenGL 着色器语言 | 写给 GPU（显卡）的编程语言，控制顶点和像素如何渲染 | 像给显卡写「怎么画」的指令——画成金属质感还是木头质感 |
| **HLSL** | High-Level Shader Language | 高级着色器语言 | 微软的 GPU 编程语言，用在 Direct3D 上 | 功能和 GLSL 一样，但语法是微软风格的，专供 D3D 使用 |
| **SPIR-V** | Standard Portable Intermediate Representation - Vulkan | Vulkan 标准中间表示 | 一种跨平台、跨语言的着色器"中间代码"格式 | 像 Java 的 .class 文件——不管用什么语言写，最终都编译成 SPIR-V 给 Vulkan 用 |
| **GLES** | OpenGL for Embedded Systems | 嵌入式 OpenGL | 简化版的 OpenGL，专门给手机/平板用的 | 像 OpenGL 的轻量版——手机芯片性能有限，用 GLES 省电省性能 |
| **MSL** | Metal Shading Language | Metal 着色器语言 | 苹果设备（iPhone/Mac）的 GPU 编程语言 | 像苹果生态的专属语言——只有苹果设备能用，但性能很好 |

## 五、引擎架构

| 缩写 | 英文全称 | 中文名称 | 大白话解释 | 打个比方 |
|:----:|---------|:--------:|-----------|---------|
| **ECS** | Entity Component System | 实体组件系统 | 一种游戏对象组织方式：实体是空壳子，往里装不同"零件"变成不同东西 | 像乐高：灰色底板是实体，积木块是组件，装轮子变汽车、装墙变房子 |
| **EnTT** | EnTT（原名） | EnTT 开源 ECS 库 | 一个高性能的开源 ECS 框架，DSEngine 直接用它的代码 | 像做菜直接用超市买的调料包——不用自己从头调配，省时间 |
| **ServiceLocator** | Service Locator（原名） | 服务定位器 | 一个"服务中心"，引擎里所有功能模块都从这里注册和获取 | 像酒店前台——你要什么服务（保洁/维修/叫醒），打电话给前台就行 |
| **RenderGraph** | Render Graph（原名） | 渲染图 | 用节点图的方式组织渲染任务的先后顺序和依赖关系 | 像工厂流水线的设计图——先装什么零件、后装什么，一目了然 |
| **FramePipeline** | Frame Pipeline（原名） | 帧流水线 | 引擎每一帧执行流程的总调度器，决定先做什么后做什么 | 像餐厅的出餐流程——接单→下单→做菜→装盘→上菜，每帧都走一遍 |
| **DAG** | Directed Acyclic Graph | 有向无环图 | 一种不会形成循环的箭头图，用来表示任务的先后依赖关系 | 像做菜流程图：洗菜→切菜→炒菜，不能反过来，也不会无限循环 |
| **UBO** | Uniform Buffer Object | 统一缓冲区对象 | 一块 GPU 内存，存放渲染时不会变化的"统一参数"（如灯光位置） | 像一张贴在墙上的操作说明书——所有工人都能看到同样的指导信息 |
| **SSBO** | Shader Storage Buffer Object | 着色器存储缓冲区对象 | 比 UBO 更灵活的 GPU 内存，可读可写，容量更大（需要 GL 4.3+） | 像一块白板——工人不仅可以看，还可以在上面写东西更新信息 |
| **DLL** | Dynamic-Link Library | 动态链接库 | Windows 上的 .dll 文件，把代码打包成独立模块，运行时按需加载 | 像工具箱里的独立工具——要用扳手才拿出来，不用就一直放着 |
| **SDK** | Software Development Kit | 软件开发工具包 | 一套完整的工具和代码库，让开发者能基于某个平台做开发 | 像乐高的乐高城市套装——包含图纸 + 特殊积木 + 工具，让你拼出特定场景 |
| **API** | Application Programming Interface | 应用程序编程接口 | 一组预先定义好的函数/接口，让不同软件之间能互相调用 | 像餐厅的菜单——你点菜（调函数），厨房做菜（执行功能），不需要知道厨房内部 |
| **EventBus** | Event Bus（原名） | 事件总线 | 引擎内部的消息广播系统，一处发事件，所有监听的地方都能收到 | 像公司的广播系统——人事部喊一声「发奖金了」，全公司都能听到 |
| **JobSystem** | Job System（原名） | 任务系统 | 把大任务拆成小任务，自动分配到多个 CPU 核心上并行执行 | 像搬家公司——一个人搬大柜子累死，分成箱子让几个人一起搬快多了 |
| **REPL** | Read-Eval-Print Loop | 交互式编程环境 | 输入一行代码立刻执行看到结果，不用写完整文件再编译 | 像计算器——按 1+1 立刻出 2，不用写一个完整的程序再运行 |

## 六、动画系统

| 缩写 | 英文全称 | 中文名称 | 大白话解释 | 打个比方 |
|:----:|---------|:--------:|-----------|---------|
| **IK** | Inverse Kinematics | 反向运动学 | 已知手/脚要放到的位置，自动算出各关节怎么弯曲 | 像伸手拿杯子——大脑自动算好肩肘腕角度，不用一个个关节去想 |
| **FABRIK** | Forward And Backward Reaching Inverse Kinematics | 前向后向迭代 IK 算法 | 一种高效的计算 IK 的算法，通过反复前后推算来快速求解 | 像用卷尺量距离——先从肩膀往前量到手，再从手往后量回肩膀，反复几次就对了 |
| **FSM** | Finite State Machine | 有限状态机 | 用「状态」+「切换条件」来控制角色行为或动画 | 像游戏角色行为图：待机→走路→跑步→跳跃，每个行为是一个"状态" |
| **LOD** | Level of Detail | 细节层次 | 远处物体用低精度渲染，近处用高精度，节省性能 | 远山只看轮廓，走近才看清石头和树——省显卡力气 |
| **Blend Tree** | Blend Tree（原名） | 混合树 | 把多个动画平滑地混合在一起，比如走路→跑步的平滑过渡 | 像 DJ 打碟把两首歌慢慢混到一起，过渡非常自然 |
| **Crossfade** | Crossfade（原名） | 交叉淡入淡出 | 一个动画淡出，另一个动画淡入，实现平滑切换 | 像视频剪辑中的淡入淡出效果——两个画面一隐一现地切换 |
| **Root Motion** | Root Motion（原名） | 根骨骼运动 | 用动画本身来控制角色移动，而不是用代码控制位移 | 像角色走路时，脚步动画自带的移动会推动身体前进，不需要额外写移动逻辑 |
| **Bone Mask** | Bone Mask（原名） | 骨骼遮罩 | 指定哪些骨骼受当前动画影响，哪些不受影响 | 像上半身射击动画 + 下半身走路动画同时播放——用遮罩只影响上半身 |
| **Morph Target** | Morph Target（原名） | 变形目标 | 通过顶点位置插值来实现面部表情等变形效果 | 像 3D 建模里做表情——从"无表情"到"微笑"，中间的所有帧自动算出来 |
| **Skinned Mesh** | Skinned Mesh（原名） | 蒙皮网格 | 模型顶点绑定到骨骼上，骨骼动，皮肤跟着动 | 像木偶戏——骨架（骨骼）动，套在骨架上的布偶衣服（网格）跟着变形 |

## 十三、CPU 指令集优化

| 缩写 | 英文全称 | 中文名称 | 大白话解释 | 打个比方 |
|:----:|---------|:--------:|-----------|---------|
| **SIMD** | Single Instruction Multiple Data | 单指令多数据 | 一条 CPU 指令同时处理多份数据，一次计算顶好几次 | 像用一排筷子同时夹一排豆子——一次干多个活，效率翻倍 |
| **SSE** | Streaming SIMD Extensions | 流式 SIMD 扩展 | Intel CPU 上的 SIMD 指令集，从 2000 年左右开始几乎所有 CPU 都支持 | 像老款加速器——大部分电脑都有，但速度一般 |
| **AVX** | Advanced Vector Extensions | 高级向量扩展 | SSE 的升级版，一次能处理更多数据，速度更快 | 像新款加速器——比 SSE 处理量大一倍，但需要较新的 CPU |
| **NEON** | ARM NEON（原名） | ARM SIMD 指令集 | 手机/平板上 ARM 芯片的 SIMD 加速指令 | 像手机的专属加速器——苹果/高通芯片都有，专门给移动设备用 |
| **FMA** | Fused Multiply-Add | 融合乘加运算 | 一条指令同时完成"乘法+加法"，比分开做更快更精确 | 像买一送一——本来要两步，现在一步完成 |
