## 一、锯齿是怎么来的？—— 数学老师 VS 显示器

### 先说结论

> **锯齿的本质是：现实中平滑的连续线条，被屏幕的像素网格"强行打码"了。**

### 用生活场景理解

想象你在方格纸上画一条斜线：

- 现实中，这条线是**平滑的**（数学老师的完美直线）
- 但方格纸只有一个个小格子，你只能在某些格子里涂色
- 涂出来的结果：一条**犬牙交错**的"阶梯"线

这就是**锯齿**的诞生过程。

### 计算机图形学的叫法

科学家给它起了个学名：**走样（Aliasing）**。意思是——**现实中的平滑信号（连续），被采样成离散的格子（离散），结果"走样"了**。

> 不光是画面边缘有锯齿，还会出现：
> - **摩尔纹** —— 远处栅栏出现奇怪的花纹
> - **闪烁** —— 细小物体在屏幕上闪来闪去
> - **车轮倒转** —— 电影里汽车轮子看起来在往回转

那怎么办？**抗锯齿（Anti-Aliasing）** 就是各种"消除锯齿"技术的总称。

## 二、SSAA —— 最暴力的抗锯齿，显卡的噩梦

SSAA 的全称是 **Super-Sampling Anti-Aliasing**（超采样抗锯齿）。

### 原理：杀鸡用牛刀

想象你拍一张照片：

1. 用 4 倍像素的超级相机拍（1920×4 × 1080×4 = **3300 万像素**）
2. 然后把这张巨幅照片**缩小**到 1920×1080
3. 缩小时把 4 个像素平均成 1 个，**锯齿就被磨平了**

### 为什么说它"暴力"？

| 倍数 | 像素数 | 性能开销 |
|:----:|:------:|:--------:|
| 1x（不开） | 1920×1080 = 207 万 | 100% |
| 2x SSAA | 3840×2160 = 829 万 | **4 倍** |
| 4x SSAA | 7680×4320 = 3317 万 | **16 倍** |

**结论：SSAA 效果好到爆炸，但显卡会直接跪下来叫你爸爸。** 即使 RTX 4090 在 4K 下开 4x SSAA，帧率也会直接掉到个位数。

### 实用价值

> **2010 年以后，已经没有游戏用 SSAA 了。** 太费了，性价比为负。

但它的思路是对的：**多采样 → 混合 → 降锯齿**。后面所有抗锯齿技术都是在这个思路基础上做优化。

## 三、MSAA —— 聪明一点的暴力法

MSAA 的全称是 **Multisample Anti-Aliasing**（多重采样抗锯齿）。

### 原理："偷懒"的艺术

MSAA 发现了一个秘密：**没必要对每个像素都做全部计算**。

看这张图：
```
一个三角形经过一个像素网格：

普通渲染（无AA）：
[■][■][ ][ ]
[■][■][ ][ ]
[ ][ ][ ][ ]

4x MSAA（每个像素内再采 4 个子点）：
每个像素内部判断子点是否在三角形内
子点命中越多 → 像素颜色越接近三角形颜色
```

### MSAA 聪明在哪？

- **不开 SSAA**：不渲染超高分辨率，只在一个**像素内部多采几个子点**
- **只对几何边缘做**：像素内部的三角形边界才需要多采，完全被覆盖的像素直接取色
- **只算深度/模板**：子点只判断"在不在三角形里"，不跑完整的像素着色器

### 性能对比

| 倍率 | VS SSAA 节省 | 画面质量 |
|:----:|:-----------:|:--------:|
| 2x MSAA | 比 2x SSAA 省 ~50% | 接近 2x SSAA |
| 4x MSAA | 比 4x SSAA 省 ~70% | 接近 4x SSAA |
| 8x MSAA | 基本够用了 | 几乎看不出锯齿 |

### MSAA 的致命伤

**MSAA 对"着色器内部锯齿"完全无效。**

什么意思？举个栗子：

```
MSAA 能处理：  三角形边缘的几何锯齿  ✅
MSAA 不能处理：纹理细节产生的摩尔纹 ❌
MSAA 不能处理：镜面高光的闪烁     ❌
MSAA 不能处理：次表面散射的噪点   ❌
```

> 因为 MSAA 只对"几何边缘"多采样，像素着色器里的复杂计算不做多重采样。现代游戏越来越依赖像素着色器做效果（PBR、SSS、反射），MSAA 的用处越来越小。

### 现状

> **2015 年以后的游戏引擎，MSAA 更多是"保留选项"。** DSEngine 默认开 4x MSAA 做基础抗锯齿，配合其他技术一起用。

## 四、FXAA —— 走捷径，最快最省

FXAA 的全称是 **Fast Approximate Anti-Aliasing**（快速近似抗锯齿）。

### 换个思路

前面的 SSAA 和 MSAA 都是在"渲染阶段"解决锯齿。FXAA 说：

> **别那么麻烦，等画面已经画好之后，我用 PS 把锯齿 P 掉不就行了？**

### FXAA 怎么做？

FXAA 是一个**后处理特效**（Post Processing），意思是：画面已经渲染完了，在最后输出前加一道工序。

它的套路：

```
Step 1：遍历每个像素，检测它周围有没有"亮度突变"
        ── 如果有，说明这里是锯齿边缘
Step 2：判断边缘的方向（横的还是竖的？朝哪个方向斜？）
Step 3：沿着边缘方向，把两边的颜色做模糊混合
        ── 锯齿的"阶梯"就被模糊掉了
```

### 效果

| 优点 | 缺点 |
|:----|:----|
| **性能极好** —— 1ms 都不到 | **画面会糊** —— 非锯齿区域也被模糊了 |
| **不挑 API** —— OpenGL/Vulkan/D3D 都能用 | **纹理细节损失** —— 远处的条纹纹理会被抹掉 |
| **管你什么类型的锯齿** —— 几何/纹理/高光一律处理 | **不开 TAA 的话，动态场景会闪烁** |
| **一行 shader 代码就能搞定** | 不能完全消除锯齿，只能"减轻" |

### 一句话总结

> **FXAA 是最省钱的抗锯齿方案：只花 5% 的性能成本，解决 50% 的锯齿问题。** 适合笔记本、集成显卡等低配机器。

## 五、TAA —— 用时间换画质的黑科技

TAA 的全称是 **Temporal Anti-Aliasing**（时间抗锯齿）。

### 核心思路

> **把每一帧之间浪费的"时间"利用起来。**

### 生活场景

你拍照时手抖了一下，拍糊了。但如果你把**连续几张拍糊的照片叠在一起平均**，结果反而比单张更清晰？

这听起来反直觉，但 TAA 就是这么干的。

### TAA 的具体做法

```
每帧要做两件事：

① 抖动（Jitter）—— 每次偏移一点点
   第 1 帧：摄像机位置偏移 (0.1, 0.1) 像素
   第 2 帧：偏移 (0.3, 0.7) 像素  
   第 3 帧：偏移 (0.5, 0.3) 像素
   ...
   Halton 序列决定偏移值（一种均匀分布的随机数）

   效果：每次锯齿的位置都不一样
         = 锯齿变成了"噪点"，而不是固定的一条线

② 混合（Blend）—— 把当前帧和历史帧混在一起
   最终颜色 = 当前帧 × 0.1 + 历史帧 × 0.9

   效果：噪点被平均掉，画面变平滑
         = TAA 的输出结果
```

### Halton 序列是什么？

TAA 里用到的这个"抖动偏移"不是随便瞎抖的。用的是 **Halton 序列**——一种"看着随机但其实很均匀"的数学序列。

```
Halton(1, base=2) = 0.5
Halton(2, base=2) = 0.25
Halton(3, base=2) = 0.75
Halton(1, base=3) = 0.333
Halton(2, base=3) = 0.667
...
```

> 用两个不同基数（比如 2 和 3）的 Halton 值作为 XY 偏移量，就能保证每帧的抖动位置均匀分布在像素内部，不扎堆。

这其实就是 DSEngine 里 `TAAPass::Halton()` 函数干的事。代码就几行，但效果暴好。

### TAA 的优点

| 优点 | 说明 |
|:----|:------|
| **效果好** | 能干掉 90% 的锯齿，包括 MSAA 处理不了的纹理摩尔纹 |
| **性能好** | 只有一次全屏的混合 + 双缓冲，比 MSAA 省多了 |
| **全面** | 几何锯齿、纹理闪烁、高光噪点全部能处理 |
| **可调节** | `blend_factor` 控制混合强度，0.05~0.2 之间调 |

### TAA 的缺点

> **鬼影（Ghosting）** —— 快速移动时，历史帧和当前帧对不上，画面出现"拖尾"。

比如：
- 快速转头 → 上一帧的墙壁残留画面在当前帧出现，像"鬼影"一样跟着
- 角色快速挥手 → 手后面有一串半透明的手影

**解决方案：**
- **Variance Clipping** —— 限制混合范围，把不正常的颜色裁掉
- **Motion Vector** —— 用运动向量做"重投影"，让历史帧和当前帧对齐（DSEngine 就传了 motion vector 纹理到 TAA Pass）
- **降低 blend_factor** —— 混合比例越低，鬼影越轻，但去锯齿效果也减弱

### 为什么 TAA 现在是标配？

> **2016 年以后，AAA 游戏几乎标配 TAA**（《守望先锋》《战地 1》《巫师 3》后期补丁都加了 TAA）。

原因很简单：
1. MSAA 处理不了 PBR 的闪烁
2. FXAA 会糊
3. TAA 效果最好、性能适中

**但 TAA 也不是完美无缺**——鬼影和动态模糊的权衡一直是图形学大佬们的研究热点。

## 六、它们谁最好？一张表说清楚

| 技术 | 全称 | 原理 | 画面 | 性能 | 缺点 | 适用于 |
|:----:|:----|:----|:---:|:----:|:----|:------|
| **SSAA** | Super-Sampling | 超高分辨率渲染再缩小 | ⭐⭐⭐⭐⭐ | ❌❌❌❌❌ 最费 | 显卡冒烟 | 离线渲染 |
| **MSAA** | Multisample | 像素内部多重采样边缘 | ⭐⭐⭐⭐ | ❌❌❌ 中等 | 对着色器锯齿无效 | 老游戏 + 基础抗锯齿 |
| **FXAA** | Fast Approximate | 后处理模糊边缘 | ⭐⭐⭐ | ❌ 最省 | 糊，纹理细节丢失 | 低配机、移动端 |
| **TAA** | Temporal | 多帧抖动+历史混合 | ⭐⭐⭐⭐⭐ | ❌❌ 较低 | 鬼影，动态模糊 | **现代游戏标配** |

### 在游戏里怎么选？

```
┌─ 显卡很好（RTX 4080+）────────────┐
│  4x TAA + 4x MSAA（叠 buff）      │  ← 目前画质天花板
└───────────────────────────────────┘
┌─ 显卡中等（RTX 3060）─────────────┐
│  TAA（不开 MSAA）                 │  ← 性价比最高
└───────────────────────────────────┘
┌─ 显卡一般（GTX 1650）─────────────┐
│  FXAA（或者 TAA 低配）            │  ← 保证帧率优先
└───────────────────────────────────┘
┌─ 集成显卡（UHD 620）─────────────┐
│  FXAA（或者关掉）                 │  ← 能跑就行
└───────────────────────────────────┘
```

## 七、DSEngine 的抗锯齿全家桶

DSEngine 目前集成了**三种**抗锯齿技术，可以叠加使用：

### 7.1 MSAA 4x —— 基础防线

引擎默认给场景渲染目标开了 **4x MSAA**：

```cpp
// frame_pipeline.cpp 第 235 行
scene_desc.msaa_samples = 4;
```

如果显卡不支持，会自动降级到 1x（不开）。**这是第一道防线，专门处理几何边缘的锯齿。**

### 7.2 FXAA —— 兜底方案

FXAA Pass 作为后处理，在 TAA 之后运行。**它的作用是兜底——TAA 没处理干净的锯齿，FXAA 再抹一把。**

```cpp
// frame_pipeline.cpp 第 891 行
registered_passes_.push_back(std::make_unique<dse::render::FXAAPass>(render_pass_context_));
```

### 7.3 TAA —— 画质担当

TAA 是 DSEngine 的**主力抗锯齿**。它在真正的渲染管线中是这样的：

```
渲染流程：
Scene Pass（带 Jitter 抖动）→ 产生带偏移的画面 → TAA Resolve（混合历史帧）→ FXAA（可选）→ 输出

TAA 内部流程：
① 用 Halton 序列算当前帧的抖动偏移（每帧都不同）
② 读取 motion vector 纹理（来自上一帧场景渲染）
③ 用 blend_factor 混合当前帧 + 历史帧
④ 把结果写入双缓冲的历史 RT（ping-pong 切换）
⑤ 下一帧重复
```

**Lua 层面可以控制：**

```lua
-- 启用 TAA（默认关闭）
entity:get_or_create(PostProcessComponent)
pp = entity:get(PostProcessComponent)
pp.enabled = true
pp.taa_enabled = true
pp.taa_blend_factor = 0.1  -- 混合强度，0.05~0.2
```

### 7.4 三者的配合

```
MSAA 4x → 先干掉几何锯齿         +   TAA → 干掉纹理/高光闪烁/剩余锯齿
                                      ↓
                             再加上 FXAA 兜底
                                      ↓
                             最终输出：平滑画面
```

> 三管齐下，目前 DSEngine 的抗锯齿水平能达到主流商业引擎 80% 的效果——但只花了约 **1/1000 的代码量**来做这件事。
>
> 缺什么？缺 TAA 的 **Variance Clipping**（防鬼影的高级策略）和更好的 Motion Vector 重投影。这些需要更复杂的 shader 代码，是未来的优化方向。

## 总结

```
锯齿 = 平滑线条被像素网格"打码"了

抗锯齿流派：
  SSAA ── 暴力渲染高分辨率再缩小     ← 效果好到爆，性能差到哭
  MSAA ── 只在像素内部多采几个点     ← 聪明点，但对现代渲染无效
  FXAA ── 画面画好后 PS 模糊锯齿     ← 最省，但会糊
  TAA  ── 每帧抖一下，多帧叠一起     ← 当今业界标配，性价比之王

DSEngine 方案：MSAA 4x ➕ TAA ➕ FXAA = 三保险
```
