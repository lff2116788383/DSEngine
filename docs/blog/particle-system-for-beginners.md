# 粒子系统——火焰、烟雾、魔法特效是怎么飞起来的？

> **灵魂拷问：游戏里放一个火球术，火焰噼里啪啦地飞出去——那是一整个特效模型，还是好多好多小东西在同时动？**
>
> 答案是**好多好多小东西**。一个火球可能有几千上万个"小点"在同时飞舞。这些小点叫做**粒子（Particle）**，而管着它们的系统就叫**粒子系统（Particle System）**。

## 一、粒子就是"很多很多小东西"——烟花、火焰、魔法都是粒子

### 什么是粒子？

**粒子（Particle）** 就是游戏里一个非常小的"点"。它通常是一个：
- 小正方形（始终面向镜头）
- 或者一个小球体
- 甚至只是一条亮线

一个粒子本身很小很不起眼，但**成千上万个粒子一起动**，就能产生惊人的视觉效果。

### 游戏里哪些效果是粒子做的？

| 效果 | 粒子数量 | 每个粒子长啥样 |
|:----|:--------|:--------------|
| 🔥 **火焰** | 几百~几千个 | 橙黄色半透明的点 |
| 💨 **烟雾** | 几百个 | 灰色半透明、逐渐变大的点 |
| ✨ **魔法/光效** | 几十~几百个 | 彩色发光的点 |
| 🎆 **烟花** | 几百~几千个 | 彩色亮点，向外飞散 |
| 💧 **雨水** | 几千个 | 白色细线 |
| 🌊 **喷泉** | 几百个 | 透明蓝色小球 |
| 🪨 **碎石** | 几十个 | 不规则的灰色小块 |
| 🦟 **萤火虫** | 几十个 | 黄绿色小光点 |

### 用生活场景理解

🎆 **烟花：**

一个烟花升空爆炸的过程：

```
烟花升空 → 一个"大粒子"飞到空中
  
爆炸 →
  一个点变成了五百个光点
  每个光点向不同方向飞散
  慢慢变暗 → 消失
  
整个过程就是"一个发射器产生了一堆粒子"
每个光点 = 一个粒子
```

### 粒子的形状

虽然叫"粒子"，但它们不一定是圆形小点。粒子通常是**始终面对摄像机的正方形**（Billboard 技术），用一张半透明的纹理贴在上面：

```
你看到的粒子效果：
  ╭──╮  ╭──╮  ╭──╮
  │🔥│  │🔥│  │🔥│  ← 每个粒子是一个小正方形
  ╰──╯  ╰──╯  ╰──╯  上面贴了火焰纹理

其实每个粒子：
  ┌──┐  ┌──┐  ┌──┐
  │  │  │  │  │  │  ← 只是正方形
  └──┘  └──┘  └──┘  但始终正对着你
```

## 二、发射器：粒子的"出生点"

### 没有发射器就没有粒子

**发射器（Emitter）** 是粒子系统的起点。它决定了：
- 粒子从哪里生出来
- 粒子初始朝哪个方向飞
- 粒子初始速度多快
- 多久产生一个新粒子

### 发射器的形状

粒子的出生位置不同，效果完全不同：

```
① 点发射器（Point Emitter）
     ● ← 所有粒子从同一个点发出
     适合：爆炸、烟花

② 圆锥发射器（Cone Emitter）
     ●
    ╱ ╲
   ╱   ╲
 粒子从圆锥内部发出
 适合：火焰喷射器、手电筒光束

③ 盒状发射器（Box Emitter）
   ┌─────┐
   │粒子 │ 粒子从盒子内部随机位置发出
   │ 从   │
   │ 这   │ 适合：下雨、下雪
   │ 里   │
   └─────┘

④ 球形发射器（Sphere Emitter）
    ╭────╮
   ╱ 粒子 ╲  粒子从球体表面或内部发出
  │ 从内部  │ 适合：爆炸冲击波、光环
   ╲      ╱
    ╰────╯
```

### 发射器的参数

```cpp
// 发射器的"设置"
emitter = {
    emissionRate: 100,      // 每秒产生 100 个粒子
    spawnShape: "cone",     // 圆锥形发射
    spawnDirection: (0,1,0),// 朝上发射
    spawnSpeed: 5.0,        // 初始速度 5 米/秒
    lifetime: 2.0,          // 粒子存活 2 秒
    startColor: (1,0.5,0),  // 初始颜色：橙色
    endColor: (0.5,0,0),    // 最终颜色：深红
    startSize: 0.1,         // 初始大小 0.1 米
    endSize: 0.5,           // 最终大小 0.5 米
}
```

## 三、粒子的生命周期：出生 → 运动 → 变化 → 死亡

### 每个粒子都有自己的"一生"

```
粒子的生命：

① 出生（Spawn）
   从发射器产生
   获得初始位置、速度、颜色、大小
   
② 运动（Update）
   每帧更新位置（速度 × 时间）
   受重力影响（火焰向上，雨水向下）
   受风力影响（烟雾被吹散）

③ 变化（Evolve）
   颜色渐变：从亮变暗、从橙变红
   大小变化：从大到小或从小到大
   透明度变化：从不透明到半透明到消失

④ 死亡（Death）
   寿命到了 → 粒子消失
   或者飞出范围 → 粒子消失
   死亡后，粒子可以被"回收"重新使用
```

### 火焰粒子的生命周期

```
火焰粒子的一生（大约 1 秒）：

起点（出生）：
  颜色：亮黄色  大小：小  透明度：不透明
  速度：向上 2 米/秒
  
第 0.2 秒：
  颜色：橙黄色  大小：变大  透明度：轻微透明
  
第 0.5 秒：
  颜色：橙色    大小：最大  透明度：半透明
  
第 0.8 秒：
  颜色：暗红色  大小：缩小  透明度：很透明
  
第 1.0 秒（死亡）：
  颜色：几乎黑色  大小：极小  透明度：完全透明
  → 消失
```

### 伪代码

```cpp
// 粒子的每帧更新
function updateParticle(particle, deltaTime):
    // ① 运动
    particle.velocity += gravity * deltaTime
    particle.velocity += wind * deltaTime
    particle.position += particle.velocity * deltaTime
    
    // ② 变化（根据年龄计算）
    age = particle.age / particle.maxAge  // 0~1
    
    // 颜色从 startColor 渐变到 endColor
    particle.color = lerp(particle.startColor, particle.endColor, age)
    
    // 大小变化
    particle.size = lerp(particle.startSize, particle.endSize, age)
    
    // 透明度：越来越透明
    particle.alpha = 1.0 - age
    
    // ③ 检查死亡
    particle.age += deltaTime
    if particle.age >= particle.maxAge:
        killParticle(particle)
```

### 粒子系统的"循环"

```
粒子系统就像一个永不停止的工厂：

  发射器 → 不断产生新粒子 → 粒子飞散变化 → 粒子死亡
             ↑                                      │
             └────────── 回收再利用 ────────────────┘

优化技巧（对象池）：
  不"删除"死亡的粒子
  而是标记为"可回收"
  发射器可以直接复用这些粒子
  避免频繁创建和销毁 → 性能更好
```

## 四、CPU 粒子 vs GPU 粒子

### CPU 粒子

**CPU 粒子**每个粒子的计算都在 CPU 上进行。

```
CPU 粒子的工作方式：

  每一帧：
    CPU 遍历所有粒子
    更新每个粒子的位置、颜色、大小
    然后把结果传给 GPU 去画

  优点：
    每个粒子可以有不同的行为
    可以和各种物理系统互动（碰撞、风力）
    
  缺点：
    粒子多了会卡（几千个以上 CPU 就扛不住了）
    因为 CPU 要逐个更新每个粒子的数据
```

### GPU 粒子

**GPU 粒子**把计算搬到了 GPU 上。

```
GPU 粒子的工作方式：

  每一帧：
    CPU 只需要告诉 GPU："从位置 A 发射粒子"
    GPU 自己计算每个粒子的运动、变化
    GPU 自己把粒子画出来

  优点：
    性能极好（几万甚至几十万个粒子毫无压力）
    GPU 天生适合做并行计算（一万个小学生）
    
  缺点：
    难以和 CPU 物理系统交互
    编程更复杂
```

### 两者对比

| 对比维度 | CPU 粒子 | GPU 粒子 |
|:--------|:--------|:--------|
| **最大粒子数** | 几千个到顶 | 几十万不是问题 |
| **性能瓶颈** | CPU 逐个更新 | GPU 并行计算，几乎无瓶颈 |
| **物理交互** | ✅ 可以和场景碰撞 | ❌ 很难检测碰撞 |
| **编程难度** | ✅ 简单 | ❌ 需要写 Compute Shader |
| **灵活性** | ✅ 每个粒子可以独一无二 | ❌ 所有粒子遵循相同规则 |
| **适用场景** | 少量特效、需要物理交互 | 大规模粒子（雨、雪、星空） |

### 游戏里实际怎么用？

```
火焰（几百个粒子）→ CPU 粒子 ✅
  因为火焰需要受风力影响
  可能需要和物体碰撞

暴雨（几万个粒子）→ GPU 粒子 ✅
  因为雨滴不需要碰撞
  只需要大量雨滴从天上掉下来

魔法特效（几百个粒子）→ CPU 粒子 ✅
  需要精确控制每个粒子的运动轨迹

星空背景（几万个星星）→ GPU 粒子 ✅
  慢慢闪烁就行，不需要物理交互
```

## 五、生活中的类比：烟花、喷泉、星空

### 类比 1：烟花 🎆

```
一个烟花弹升空（发射器）
爆炸（同时产生几百个粒子）
每个光点向外飞散（粒子运动）
光点慢慢变暗消失（粒子生命周期）

你看到的美轮美奂的烟花
就是一个"粒子系统"的完美例子
```

### 类比 2：喷泉 ⛲

```
喷泉底部：水从几个喷嘴喷出（发射器）
水柱上升：水流稳定的轨迹（粒子流）
水花四溅：水柱到达最高点后散开（粒子运动）
水滴落下：受重力影响掉回水池（粒子生命周期）

如果关闭喷泉 → 发射器停止工作
水池恢复平静 → 所有粒子死亡
```

### 类比 3：星空中的星星 ✨

```
冬天的夜晚仰望星空：

那些星星：
  不是所有星星都一样亮
  有些闪烁、有些稳定
  有些大、有些小
  
粒子系统的星星：
  每个星星 = 一个粒子
  亮度 = 透明度
  大小 = 粒子大小
  闪烁 = 透明度随时间变化
```

## 六、对比表格：粒子 vs 普通模型渲染

| 对比维度 | 普通 3D 模型渲染 | 粒子渲染 |
|:--------|:---------------|:--------|
| **数量** | 几个到几百个 | 几百到几十万个 |
| **每个物体的控制** | 位置、旋转、缩放 | 位置、速度、颜色、大小、透明度…… |
| **碰撞检测** | ✅ 需要 | ❌ 通常不需要 |
| **光照计算** | ✅ 完整的 PBR | ❌ 自己发光，不依赖场景光照 |
| **绘制方式** | 逐物体 Draw Call | 批量绘制，一个 Draw Call 画所有粒子 |
| **性能关键点** | GPU 渲染（三角形数量） | CPU 更新（粒子数量） |
| **实际用途** | 角色、建筑、道具 | 火焰、烟雾、魔法、雨水、火花 |

### 一个简单的判断方法

```
这个东西在游戏里：

有清晰的形状 → 模型（角色、建筑、武器）
没有固定形状 → 粒子（火焰、烟雾、光效）

会动的 → 可能是模型（动画）或者粒子
大量重复的 → 大概率是粒子
```

## 七、一句话总结

> **粒子系统就是"用成千上万个小东西制造视觉魔法"——发射器决定粒子从哪来，每帧更新决定粒子怎么飞，颜色/大小/透明度的渐变决定粒子怎么变化，寿命到了就消失。从火焰到烟花，从暴雨到魔法，游戏里那些最炫酷、最没有固定形状的效果，基本都是粒子系统在干活。**
>
> **下次在游戏里放一个火球术，看着漫天飞舞的火星——记住，每一粒火星都是一个独立的"粒子"，正在走完它短暂而灿烂的一生。** 🔥
