# Bloom——为什么亮的东西会发光？

> **灵魂拷问：你有没有发现，游戏里的太阳、灯管、霓虹灯，看起来像是在"发光"——但你的显示器明明只是一个平面，怎么可能真的发光？**
>
> 没错，显示器确实不会发光。你看到的"发光"效果，其实是游戏引擎跟你玩的一个把戏——它叫 **Bloom（泛光）**，是后处理特效里最常用的一个。

## 一、人眼对亮部有"光晕"感——Bloom 模拟这个

### 为什么会有 Bloom？

先搞明白一个问题：**现实中，亮的东西周围真的有光晕吗？**

答案是：**有，但不是你想的那样。**

```
现实中的"光晕"是怎么来的：
  
  你的眼球里面有液体（房水和玻璃体）
  这些液体不是完全透明的
  
  当强光进入眼睛：
    一部分光直接穿过 → 在视网膜上成像
    另一部分光在眼球内部散射 → 在视网膜上形成模糊的光晕
    
  这就是你看到"亮的东西周围有一圈光晕"的原因
```

### Bloom 模拟的就是这个现象

> **Bloom 就是在游戏画面里，给亮度超过一定值的区域加一圈模糊的光晕，模拟眼睛里的散射效果。**

### 用生活场景理解

🌞 **看太阳的感觉：**

你盯着太阳看（别真看，伤眼睛）：
- 太阳本身：非常亮
- 太阳周围：有一圈模糊的光晕
- 闭上眼睛：还能看到残留的光斑

Bloom 做的是：
- 找到画面里"特别亮"的区域（太阳、灯、反光）
- 给它们周围加一圈"光晕"
- 让它们看起来像"真的在发光"

### Bloom 在哪用？

| 场景 | 效果 |
|:----|:-----|
| 🏙️ 城市夜景 | 霓虹灯、路灯、车灯——都有了"发光"感 |
| ☀️ 阳光 | 太阳周围有光晕，更有"刺眼"感 |
| 💥 爆炸 | 爆炸中心亮到模糊——更有冲击力 |
| 🔦 手电筒 | 光束的亮部有光晕——更真实 |
| ✨ 魔法特效 | 魔法光芒——更炫酷 |

## 二、第一步：把画面里的亮部提取出来

### Bloom 的第一步

Bloom 是一个**后处理特效**——也就是说，场景已经渲染完了，现在要对最终的画面做处理。

第一步很简单：**把亮的部分挑出来。**

### 怎么做？

```
原始画面：
  ┌──────────────────────────┐
  │  暗色背景                 │
  │    ★ 太阳（非常亮）       │
  │      ★ 星星（微亮）      │
  │    □ 窗户（中等亮）       │
  │  暗色背景                 │
  └──────────────────────────┘

设定一个"亮度阈值"：
  亮度 > 0.8 → 提取（太阳）
  0.5 < 亮度 < 0.8 → 忽略（窗户）
  亮度 < 0.5 → 忽略（背景、星星）
  
提取结果：
  ┌──────────────────────────┐
  │  全黑                    │
  │    ★ 太阳（保留原色）    │
  │  全黑                    │
  │  全黑                    │
  │  全黑                    │
  └──────────────────────────┘
```

### 伪代码

```cpp
// 提取亮部
for each pixel in 画面:
    // 算这个像素的亮度
    brightness = (pixel.r * 0.299 + pixel.g * 0.587 + pixel.b * 0.114)
    
    // 超过阈值的才保留
    if brightness > threshold:
        brightPass[pixel] = pixel.color  // 保留颜色
    else:
        brightPass[pixel] = black        // 变黑
```

### 阈值调高调低的效果

| 阈值 | 效果 | 适用场景 |
|:----|:-----|:--------|
| 0.9 | 只有最亮的东西才有光晕 | 阳光直射、强光源 |
| 0.7 | 中等亮度的东西也有 | 霓虹灯、发光材质 |
| 0.5 | 大部分亮的东西都有 | 卡通风格、梦幻效果 |
| 0.3 | 连灰色的都算亮 | 闪瞎眼，慎用 |

## 三、第二步：不断缩小再放大——产生模糊

### 为什么需要模糊？

上一步提取出来的"亮部"是这样的：
- 太阳：一个清晰的圆形
- 周围：全黑

直接把清晰的太阳和原图叠加——**看起来就像在画面上贴了一张贴纸，根本不像是"发光"。**

所以要把它**变模糊**——模糊了才有"光晕扩散"的感觉。

### 如何模糊？——降采样 + 升采样

不用复杂的模糊算法，Bloom 用了一种非常巧妙的方法：

> **先把图片缩小（降采样），再放大回原尺寸（升采样）——缩小时会丢失细节，放大时像素之间会"插值"过渡，结果就是：图片变模糊了。**

```
原始提取的亮部（高清）：
  ┌────────────────────┐
  │  ██                │
  │  ████  太阳        │
  │  ██                │
  └────────────────────┘

缩小 2 倍（降采样）：
  ┌────────────┐
  │   ████     │  ← 太阳的细节丢失了
  │   ████     │     边缘开始模糊
  └────────────┘

缩小 4 倍（再降采样）：
  ┌────────┐
  │  ████  │  ← 更模糊了
  │  ████  │
  └────────┘

放大回原尺寸（升采样）：
  ┌────────────────────┐
  │  ████████████      │  ← 边缘柔和过渡
  │  ████████████      │     这就是光晕的效果
  │  ████████████      │
  └────────────────────┘
```

### 为什么要多次降采样？

Bloom 通常会做 **3~5 次降采样**，而不是一次到位：

```
第 0 层（原尺寸）：提取的亮部
     ↓ 缩小 2 倍
第 1 层（1/2 尺寸）：稍微模糊
     ↓ 缩小 2 倍
第 2 层（1/4 尺寸）：更模糊
     ↓ 缩小 2 倍
第 3 层（1/8 尺寸）：非常模糊
     ↓ 缩小 2 倍
第 4 层（1/16 尺寸）：超级模糊

然后从第 4 层到第 0 层，逐层放大叠加：
  第 4 层 + 第 3 层 + 第 2 层 + 第 1 层 = 最终光晕
```

**这样做的好处：**

```
单次模糊：只能产生一种模糊程度的光晕
          看起来不自然

多层模糊：近处清晰的光晕 + 远处模糊的光晕
          叠加在一起 → 非常自然的光晕效果
          像真实眼睛里的散射
```

### 为什么叫"高斯"？

你可能听说过**高斯模糊（Gaussian Blur）**。降采样本身就会产生高斯模糊的效果——因为在缩放过程中，像素之间的颜色会按照"高斯分布"混合在一起。

不需要懂高斯分布是什么，你只需要知道：

> **降采样 + 升采样 = 自带模糊效果**
> **多次降采样 + 多次升采样 = 多层次自然光晕**

## 四、第三步：把模糊的亮部和原图叠加

### 最后一步

现在你有：
1. **原图**——正常的场景渲染结果
2. **模糊的亮部**——多层次的光晕图

最后一步：**把两者叠在一起。**

```
原图（A）：
  ┌──────────────────────────┐
  │  暗色背景，太阳清晰可见   │
  │  ★                        │
  │                           │
  └──────────────────────────┘
  
模糊亮部（B）：
  ┌──────────────────────────┐
  │  光晕（太阳周围模糊一片） │
  │  ████████████             │
  │  ████████████             │
  └──────────────────────────┘

A + B = 最终画面：
  ┌──────────────────────────┐
  │  暗色背景                 │
  │  ★ 太阳清晰               │
  │  ██████ 太阳周围有光晕    │ ← Bloom 效果完成！
  └──────────────────────────┘
```

### 最终效果

```
无 Bloom：
  ┌────────────────┐
  │                │
  │    ★           │  太阳只是一个亮色的圆形
  │                │  看起来像"画"上去的
  │                │
  └────────────────┘

有 Bloom：
  ┌────────────────┐
  │   ████████     │
  │   ██★███      │  太阳周围有光晕扩散
  │   ████████     │  看起来像"真的在发光"

```

## 五、生活中的类比：手机拍夜景、阳光下的灯

### 类比 1：手机拍夜景 📱

你拿手机拍夜景：
- 路灯在照片里：**周围有一圈光晕**
- 这不是路灯真的在发光，而是**手机镜头内部的散射**
- Bloom 就是模拟这个效果

```
手机夜景照片 = 原始画面 + 镜头散射
Bloom 效果 = 原始画面 + 模拟散射
```

### 类比 2：阳光下的灯 💡

白天看一盏灯：
- 灯是亮的，但周围没有光晕
- 因为环境光太强，你的瞳孔缩小了

晚上看同一盏灯：
- 灯非常亮，周围有明显的光晕
- 因为环境光暗，你的瞳孔放大了
- 更多光进入眼睛，散射更明显

**所以 Bloom 在夜景里效果最明显，白天几乎看不出来。**

### 类比 3：毛玻璃后的灯光

```
你透过毛玻璃看后面的灯：
  灯光 → 毛玻璃 → 你的眼睛
               
不是"灯"在发光
而是"光透过毛玻璃"扩散了

Bloom 就是这个"毛玻璃"效果
```

## 六、对比表格：有 Bloom vs 无 Bloom

| 对比方面 | 无 Bloom | 有 Bloom |
|:--------|:--------|:--------|
| **灯/太阳** | 一个清晰的亮色圆形 | 周围有光晕，像在发光 |
| **真实感** | ⭐⭐⭐ "画"在屏幕上的 | ⭐⭐⭐⭐⭐ "真实发光" |
| **氛围感** | 平淡，缺少冲击力 | 强烈的视觉冲击 |
| **性能影响** | ✅ 零开销 | ⚠️ 3~5 次降采样 + 叠加，但现代显卡无压力 |
| **适用场景** | 写实但不需要强调光源 | 夜景、科幻、魔法、任何需要视觉冲击的场景 |

### 游戏里没 Bloom 会怎样？

```
场景：赛博朋克风格的城市夜景

❌ 关闭 Bloom：
  霓虹灯 → 清晰的彩色条带
  广告牌 → 清晰的发光文字
  感觉 → 像 P 上去的贴纸

✅ 开启 Bloom：
  霓虹灯 → 彩色的光晕弥散
  广告牌 → 文字周围有辉光
  感觉 → 身临其境的赛博城市
```

### Bloom 调过头会怎样？

```
适度 Bloom：
  ┌────────────────┐
  │  自然的辉光     │  ✅ 好看
  └────────────────┘

过度 Bloom：
  ┌────────────────┐
  │  整个画面白茫茫  │  ❌ 闪瞎眼
  └────────────────┘
  
就像做菜放盐：
  适量 → 提味 ✅
  过量 → 齁死 ❌
```

## 七、一句话总结

> **Bloom 就是给画面里"亮的地方"加一圈模糊的光晕——先提取亮部，再反复缩小放大把它们弄模糊，最后叠回原图。它模拟的是光线在人眼球里散射的现象，让你觉得游戏里的灯和太阳"真的在发光"。**
>
> **下次在游戏里看到霓虹灯闪耀的城市夜景，记住：那不是真的在发光，是你的显卡在跟你玩视觉魔术。** ✨
