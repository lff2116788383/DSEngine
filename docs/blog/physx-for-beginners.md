## 一、物理引擎的"三步曲"：检测碰撞 → 算力 → 更新位置

### 物理引擎每帧都在做什么？

一个物理引擎（比如 PhysX）每一帧都在重复做三件事：

```
每帧（1/60 秒内要完成）：
  
  Step 1：检测碰撞（Collision Detection）
    └─ 哪些物体之间"碰到"了？
    └─ 碰到多深？在哪个位置碰的？
  
  Step 2：计算力（Force Calculation）
    └─ 根据碰撞，算出每个物体应该受到多大的力
    └─ 还要算重力、摩擦力、弹力……
  
  Step 3：更新位置（Integration）
    └─ 根据算出来的力，更新每个物体的位置和速度
    └─ "这个桶刚才在 (0,5,0)，现在应该在 (0,4.9,0)"
```

### 用生活场景理解

🎱 **打台球：**

```
Step 1（检测碰撞）：
  白球撞到了 8 号球
  撞击点：x=1.5, y=2.0
  撞击深度：0.1mm（两个球接触了）
  
Step 2（算力）：
  白球速度：2m/s → 动量传递
  8 号球受到：xxx 牛顿的力
  白球受到：-xxx 牛顿的反作用力
  球桌摩擦力：使球减速
  
Step 3（更新位置）：
  白球新位置：撞后反弹，减速
  8 号球新位置：被撞飞，沿直线滚动
  两个球都在减速（摩擦力）
```

### 伪代码

```cpp
// 物理引擎每帧的"三步曲"
void physicsUpdate(float deltaTime):
    // Step 1：检测碰撞
    collisionPairs = detectAllCollisions(allObjects)
    
    // Step 2：计算力
    for each collision in collisionPairs:
        force = computeCollisionForce(collision)
        collision.objectA.applyForce(force)
        collision.objectB.applyForce(-force)
    
    // Step 3：更新位置
    for each object in allObjects:
        // 算上重力
        object.applyForce(GRAVITY * object.mass)
        // 更新速度和位置
        object.velocity += object.force / object.mass * deltaTime
        object.position += object.velocity * deltaTime
```

## 二、刚体 vs 软体 vs 布料：不同材质的物理模拟方式

### 刚体（Rigid Body）—— 硬邦邦的东西

**什么是刚体：** 不会变形的物体——石头、金属、桌椅、墙壁。

```
刚体的特点：
  ┌────────────────────┐
  │  受到力的作用       │
  │  → 整体移动或旋转   │
  │  → 但形状不变       │
  │                     │
  │  例子：             │
  │  石头掉地上         │
  │  → 弹起来，但不变形 │
  └────────────────────┘
```

**计算量：** 最低。只需要算 6 个数值（3 个位置 + 3 个旋转）。

### 软体（Soft Body）—— 会变形的东西

**什么是软体：** 会变形但会恢复原状——果冻、轮胎、橡皮球。

```
软体的特点：
  ┌────────────────────┐
  │  受到力的作用       │
  │  → 形状发生改变      │
  │  → 然后慢慢恢复      │
  │                     │
  │  例子：             │
  │  果冻掉地上         │
  │  → 先压扁，再弹回   │
  └────────────────────┘
```

**计算量：** 中高。要把物体离散成很多小点，算每个点的受力。

### 布料（Cloth）—— 软趴趴的东西

**什么是布料模拟：** 没有固定形状，靠"编织结构"维持——衣服、旗帜、窗帘。

```
布料的特点：
  ┌────────────────────┐
  │  由很多小点组成      │
  │  小点之间有"线"连接  │
  │  线可以拉伸但有限度  │
  │                     │
  │  例子：             │
  │  角色跑动时         │
  │  披风跟着飘动       │
  └────────────────────┘
```

**计算量：** 高。每个点受重力、风力、和其他点的拉力。

### 三种物体的对比

| 类型 | 会变形吗？ | 会恢复吗？ | 性能开销 | 游戏例子 |
|:----|:---------|:---------|:--------|:--------|
| 🪨 **刚体** | ❌ 不变 | — | 低 | 石头、桌子、金属 |
| 🍮 **软体** | ✅ 会变形 | ✅ 会恢复 | 中高 | 果冻、轮胎、橡皮 |
| 👕 **布料** | ✅ 会变形 | ❌ 不恢复 | 高 | 衣服、旗帜、窗帘 |

### 用生活场景理解

**刚体 = 石头**
- 你踢一块石头 → 石头飞出去 → 形状不变

**软体 = 海绵球**
- 你捏一个海绵球 → 球扁了 → 松手又弹回原样

**布料 = 毛巾**
- 你甩一条毛巾 → 毛巾在空中飘 → 停下来后耷拉着

## 三、碰撞体类型：盒子、球体、胶囊、网格——精度 vs 性能的取舍

### 什么是碰撞体？

在物理引擎里，每个物体需要有一个"碰撞形状"——告诉引擎"我这个物体的边界在哪"。

但不可能用真实的模型网格做碰撞检测（太慢了），所以要用**简化形状**。

### 四种常见的碰撞体

```
① 盒子（Box Collider）

  ┌──────┐
  │      │  最简单的碰撞体
  │      │  适合：墙壁、地板、箱子
  │      │  性能：⭐⭐⭐⭐⭐（最快）
  └──────┘

② 球体（Sphere Collider）

   ╭────╮
   ╰────╯  最省性能的碰撞体
            适合：球、子弹、角色（简化）
            性能：⭐⭐⭐⭐⭐（最快）

③ 胶囊（Capsule Collider）

  ╭────╮
  │    │  一个圆柱 + 两个半球
  │    │  适合：角色（人形）
  ╰────╯  性能：⭐⭐⭐⭐（很快）

④ 网格（Mesh Collider）

  ╱╲╱╲╱╲
  ╲╱╲╱╲╱  用模型的真实网格
  ╱╲╱╲╱╲  适合：复杂地形、大型物体
            性能：⭐⭐（较慢）
```

### 精度 vs 性能

```
高精度 ←─────────────────→ 高性能
    网格      胶囊    盒子/球体
    
    精细          快速
    但慢          但粗糙
```

### 游戏里的实际选择

```
角色 → 胶囊体
  ╭────╮
  │    │  ✅ 适合人形
  ╰────╯  ✅ 不会卡在楼梯上
           ❌ 但不够精细（手指、脚趾没法算）

子弹 → 球体
  ╭╮   ✅ 快速检测
  ╰╯   ❌ 长条形的子弹需要多个球体组合

地面 → 网格
  ╱╲╱╲  ✅ 精确匹配地形
  ╲╱╲╱  ❌ 性能开销大，但地形静止不动，可以优化
```

### 用生活场景理解

就像**快递打包**：
- 球体 = 气泡膜包一个球 → 最省材料
- 盒子 = 方形纸箱 → 适合大多数物品
- 胶囊 = 圆柱形包装 → 适合瓶子
- 网格 = 定制的泡沫内衬 → 最贴合但最贵

## 四、关节：铰链、弹簧、滑轨——物体之间的连接方式

### 什么是关节？

关节（Joint）是物理引擎里的"连接器"——**把两个物体连接起来，限制它们的相对运动。**

### 三种最常见的关节

**① 铰链关节（Hinge Joint）—— 像门的合页**

```
  门 ←── 铰链 ──→ 门框
           │
           │ 只能绕一个轴旋转
           │ 不能平移
           │
  例子：门、盖子、钟摆
```

**② 弹簧关节（Spring Joint）—— 像橡皮筋**

```
  物体 A ── 弹簧 ── 物体 B
           │
           │ 两个物体之间有弹性力
           │ 拉得越远，弹力越大
           │
  例子：减震器、弹射器、荡秋千
```

**③ 滑轨关节（Slider/Prismatic Joint）—— 像抽屉**

```
  抽屉 ←── 滑轨 ──→ 柜子
           │
           │ 只能沿一个方向滑动
           │ 不能旋转
           │
  例子：抽屉、电梯、伸缩臂
```

### 游戏里关节的用途

| 关节类型 | 游戏用例 | 效果 |
|:--------|:--------|:-----|
| 🔩 **铰链** | 门、宝箱盖、跷跷板 | 沿着轴转动 |
| 🪢 **弹簧** | 车辆悬挂、蹦床、弹弓 | 有弹性的连接 |
| 📏 **滑轨** | 电梯、抽屉、伸缩桥 | 沿直线滑动 |
| 🔗 **固定** | 焊接两个物体 | 两个物体变成一个 |
| ⛓️ **链条** | 锁链、铁索桥 | 多个铰链串联 |

## 五、布娃娃：角色死亡时的物理模拟

### 什么是布娃娃？

玩过《荒野大镖客》或者《GTA》吗？角色死的时候，身体会"软掉"，像布娃娃一样摔在地上——这就是**布娃娃物理（Ragdoll Physics）**。

### 布娃娃的原理

```
角色活着时：
  ┌────────────────────┐
  │  骨骼动画控制身体    │
  │  手怎么动、腿怎么摆  │
  │  都由动画决定        │
  └────────────────────┘

角色死亡时：
  ┌────────────────────┐
  │  关闭动画控制       │
  │  把身体各部位换成    │
  │  物理引擎的"刚体"    │
  │                     │
  │  头部 → 球体        │
  │  躯干 → 盒子        │
  │  上臂 → 胶囊        │
  │  前臂 → 胶囊        │
  │  ……然后用"关节"连接  │
  └────────────────────┘
```

### 布娃娃的组成

```
布娃娃结构：
  
      头部（球体）
        │ 脖子（铰链关节）
      躯干（盒子）
     ╱  │  ╲
  左臂  右臂  左腿  右腿
（胶囊）（胶囊）（胶囊）（胶囊）
  │      │      │      │
 肘关节  肘关节  膝关节  膝关节
  │      │      │      │
 左前臂  右前臂  左小腿  右小腿
```

### 布娃娃的效果

```
没有布娃娃：
  角色死的时候
  → 播放一个"死亡动画"
  → 每次死都是一样的姿势
  → 从悬崖掉下去也是同样的动作
  
  看起来不自然 😐

有布娃娃：
  角色死的时候
  → 物理引擎接管身体
  → 从悬崖掉下去 → 手脚乱舞，真实下坠
  → 被车撞 → 身体被撞飞，撞到墙上软下来
  → 滚下楼梯 → 身体真实地翻滚
  
  每次死都不一样！😮
```

## 六、对比表格：各种物理特性

| 物理特性 | 是什么 | 游戏例子 | 性能开销 |
|:---------|:------|:--------|:--------|
| 🪨 **刚体** | 不变形的物体 | 石头、武器、车辆 | ⭐ 最低 |
| 🍮 **软体** | 会变形并恢复 | 果冻、轮胎 | ⭐⭐⭐ 中高 |
| 👕 **布料** | 柔软可飘动 | 衣服、旗帜 | ⭐⭐⭐⭐ 高 |
| 💥 **粒子** | 大量小物体 | 火花、碎片、烟雾 | ⭐⭐～⭐⭐⭐⭐ |
| 💀 **布娃娃** | 角色死亡物理 | 尸体倒地 | ⭐⭐⭐ 中 |
| 🌊 **流体** | 水、岩浆 | 河流、水坑 | ⭐⭐⭐⭐⭐ 极高 |

### 为什么不是所有游戏都用满级物理？

```
性能预算：
  一帧只有 16ms（60fps）
  
  物理引擎占了：3~5ms
  → 刚体碰撞（重头戏）
  → 布料模拟（可选）
  → 布娃娃（只在死亡时）
  
  剩下的给：渲染、逻辑、AI……
  
所以游戏会根据需要"精打细算"：
  ┌────────────────────────┐
  │ AAA 大作               │
  │ → 刚体 + 布料 + 布娃娃  │
  │ → 精细碰撞             │
  ├────────────────────────┤
  │ 手机游戏               │
  │ → 只有刚体             │
  │ → 简化碰撞             │
  │ → 甚至没有物理引擎      │
  ├────────────────────────┤
  │ 物理模拟游戏（《糖豆人》）│
  │ → 刚体优先             │
  │ → 大量碰撞             │
  │ → 牺牲画质换物理        │
  └────────────────────────┘
```

## 七、一句话总结

> **PhysX 物理引擎就是游戏世界里的"物理老师"——每帧都在偷偷计算"谁撞了谁、受了多少力、应该往哪飞"。它用刚体模拟石头、用布料模拟披风、用布娃娃模拟角色死亡，让游戏世界看起来像真实世界一样有物理规则。**
>
> **下次在游戏里踢飞一个桶，看着它撞翻三个敌人——那不是游戏设计好的，是物理引擎在实时算出来的。** 🎯
