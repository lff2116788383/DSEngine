# 光照模型进化史——从"看起来像"到"看起来就是"

> **灵魂拷问：1980 年代的游戏和今天的 3A 大作，最大的区别在哪？**
>
> 不是分辨率，不是模型精细度——而是**光**。30 年前的游戏里，物体像塑料一样"假"；今天 PBR 渲染的物体几乎能以假乱真。这中间发生了什么？答案就是**光照模型**的进化——一个从"凑合着看"到"严格模拟物理"的过程。

## 一、Lambert（漫反射模型）—— 光照模型的"祖师爷"

### 谁提出了 Lambert？

**Johann Heinrich Lambert**——18 世纪的瑞士数学家。他 1760 年提出了一条关于光反射的定律，**200 多年后**被用在了计算机图形学里。

### Lambert 的核心思想

> **一个表面看起来有多亮，取决于它"正对"光源的程度。**

### 用生活场景理解

☀️ **晒太阳：**

```
正午（正对太阳）：
  太阳在你的头顶正上方
  阳光直射你的头顶
  → 你感觉最热、最亮
  → 光线垂直射入

黄昏（斜对太阳）：
  太阳在地平线附近
  阳光斜着照在你身上
  → 你感觉没那么热、没那么亮
  → 光线是斜着射入的
```

这个现象用数学描述就是：

```
亮度 = 光线方向 与 表面朝前方向 的"对齐程度"

完全对齐（直射）→ 最亮
完全垂直（光线贴着表面滑过）→ 完全不亮
中间 → 逐渐过渡
```

### Lambert 的公式直观理解

```
表面完全正对光源 → 亮度 = 100%
表面倾斜 45° → 亮度 ≈ 70%
表面倾斜 60° → 亮度 ≈ 50%
表面倾斜 90° → 亮度 = 0%（光线贴着表面滑过）
```

### Lambert 的效果

```
Lambert 光照效果：

  🌞              🌞              🌞
   │               │               │
   ▼               ▼               ▼
┌──────┐      ┌──────┐      ┌──────┐
│ 最亮  │      │ 中等  │      │ 最暗  │
│ 正对  │      │ 斜对  │      │ 侧面  │
└──────┘      └──────┘      └──────┘
```

### Lambert 的问题

```
问题：没有高光

看一个 Lambert 渲染的球体：
  
    ╭────╮
   ╱      ╲
  │  均匀   │  ← 所有方向看起来都一样亮
  │  过渡   │     没有"高光点"
   ╲      ╱
    ╰────╯
    
  看起来像：一个粗糙的石膏球 ✅
  看起来不像：一个光滑的陶瓷球 ❌
  看起来绝不像：金属球 ❌❌❌
```

**Lambert 只能模拟"完全没有高光的粗糙表面"。**

## 二、Phong（高光模型）—— 加了"高光"就变立体了

### 谁提出了 Phong？

**Bui Tuong Phong**——越南裔计算机科学家。1975 年，他在博士论文里提出了 Phong 光照模型——**在 Lambert 的基础上增加了高光（Specular）**。

### Phong 的创新

Phong 说：**光不只是"漫反射"（光散到各个方向），还有"镜面反射"（光集中反射到一个方向）。**

```
Phong 光照的两部分：

① 漫反射（Diffuse）= Lambert 的部分
   光打到表面 → 向四面八方均匀反射
   决定了物体"本身"的颜色
  
② 高光（Specular）= Phong 加的部分
   光打到表面 → 主要朝一个方向反射
   决定了物体"光泽"的亮斑
```

### 用生活场景理解

🍎 **一个苹果的光泽：**

```
看一个打了蜡的红苹果：

漫反射（Lambert）：
  苹果是红色的
  不论从哪个角度看，红色都在
  → 这是"物体本身的颜色"

高光（Phong）：
  苹果上有一块白色的亮斑
  你换个角度，亮斑的位置变了
  → 这是"光源的反射"
```

### Phong 高光的参数

```
Phong 高光有两个关键参数：

① 高光强度（Specular Intensity）
   控制高光有多亮
   塑料 → 高光强
   木头 → 高光弱

② 光泽度（Shininess）
   控制高光范围有多大
   光滑 → 高光范围小（一个点）
   粗糙 → 高光范围大（一片）
```

### Phong 的效果

```
Phong 光照效果：

  ╭────╮          ╭────╮          ╭────╮
 ╱  ●   ╲        ╱  ●   ╲        ╱  ●   ╲
│  白点  │      │ 白点  │      │ 白点  │
 ╲      ╱        ╲      ╱        ╲      ╱
  ╰────╯          ╰────╯          ╰────╯
  
  亮闪闪 ✨      中等光泽          哑光

Shininess=高   Shininess=中   Shininess=低
```

### 为什么 Phong 是一个大进步？

```
没有 Phong（只有 Lambert）：
  物体像石膏、像粉笔
  没有"立体感"
  
有 Phong：
  物体有"光泽感"
  看起来更真实
  不同材质之间的区别更明显
```

## 三、Blinn-Phong（更快的高光）—— 性能的胜利

### 谁提出了 Blinn-Phong？

**Jim Blinn**——另一位计算机图形学大佬。1977 年，他对 Phong 模型做了改进——在保持视觉效果几乎不变的前提下，**让计算速度更快**。

### Phong 的问题

```
Phong 高光的计算需要：
  ① 反射光方向（根据光源方向和法线算）
  ② 观察方向（看的人站在哪）
  ③ 算反射方向和观察方向的夹角

问题出在第①步：
  算"反射方向"需要比较复杂的数学
  在 1977 年，电脑慢得要死
  每次计算都宝贵
```

### Blinn-Phong 的优化

Blinn 发现了一个数学技巧：

> **不需要真的算反射方向。改用"半角向量（Half Vector）"——光源方向和观察方向的中间方向。**

```
Phong 的算法：
  光源方向 → 算反射方向 → 和观察方向比较
  
Blinn-Phong 的算法：
  光源方向 + 观察方向 → 算半角向量 → 和法线比较
  
  半角向量 = (光源方向 + 观察方向) ÷ 2
```

### 为什么 Blinn-Phong 更快？

```
Phong：
  需要计算"反射向量"——涉及多次乘法和除法
  在 1977 年的电脑上 → 慢

Blinn-Phong：
  只需要把"光源方向"和"观察方向"加起来
  再做一次归一化
  在 1977 年的电脑上 → 快很多
```

### Blinn-Phong 的视觉效果

```
对眼睛来说，Blinn-Phong 和 Phong 几乎一样：

Phong 的高光：                    Blinn-Phong 的高光：
  ╭────╮                          ╭────╮
 ╱  ●   ╲                        ╱  ●   ╲
│  白点  │                      │  白点  │
 ╲      ╱                        ╲      ╱
  ╰────╯                          ╰────╯

区别非常小
但 Blinn-Phong 快了 30%~50%
```

### Blinn-Phong 时代的游戏

```
Blinn-Phong 统治了游戏界大约 20 年（1995~2015）：

  1996 年：《古墓丽影》
  2001 年：《光环》
  2004 年：《魔兽世界》
  2007 年：《使命召唤 4》
  2011 年：《上古卷轴 5：天际》
  2013 年：《最后生还者》

绝大多数 2000~2015 年的 3D 游戏
用的都是 Blinn-Phong 光照模型
```

### Blinn-Phong 的问题

```
Blinn-Phong 虽然快，但有一个根本问题：
  它不基于物理

  "高光强度"和"光泽度"两个参数
  不是真实的物理属性
  是"调出来好看的数值"
  
  结果：
    同样的参数，换个光照环境
    看起来就不自然了
    美术要重新调参数
```

## 四、PBR GGX（真实物理）—— 最终形态

### PBR GGX 是什么？

**PBR（Physically Based Rendering）** 我们已经在第 1 篇里详细讲过。而 **GGX** 是 PBR 里用来计算高光（镜面反射）的具体公式——就像 Phong 是一个公式，GGX 是另一个更准的公式。

### GGX 比 Phong 好在哪？

```
用 Phong 高光渲染金属球：
  
  光照从一个方向来
  → 高光是一个完美的圆形亮斑
  → 但是，真实金属的高光不是正圆形！
  
用 GGX 高光渲染金属球：
  
  光照从一个方向来
  → 高光会根据观察角度"拉伸"
  → 边缘的高光被拉长、压扁
  → 和真实金属完全一致
```

### 用生活场景理解

🥄 **看一个不锈钢勺子：**

```
你用手机闪光灯照勺子：

转动勺子：
  高光点会"拉长"和"变形"
  不是完美的圆形
  这是因为勺子表面的微表面结构
  
  GGX 模拟的就是这个"微表面"效果
  它认为：微观上，表面有无数个小镜面
  每个小镜面的朝向不同
  所以高光看起来是"拉伸"的
```

### PBR GGX 的物理参数

```
PBR GGX 的参数：
  
  ① 粗糙度（Roughness）
     控制高光是清晰还是模糊
     0.0 = 镜面（倒影清晰）
     1.0 = 粗糙（倒影模糊）
     
  ② 金属度（Metallic）
     控制物体是金属还是非金属
     0.0 = 非金属（木头、塑料）
     1.0 = 金属（铁、金、铜）
     
  ③ 反射率（F0）
     控制表面的基础反射率
     非金属 ≈ 2%~5%
     金属 ≈ 70%~100%
```

### PBR 时代的游戏

```
2015 年以后，几乎所有 3A 大作都转向了 PBR：

  2015 年：《巫师 3》
  2016 年：《战地 1》
  2017 年：《地平线：零之曙光》
  2018 年：《荒野大镖客 2》
  2020 年：《赛博朋克 2077》
  2023 年：《心灵杀手 2》

为什么？
  因为"物理正确"的渲染
  在任何光照环境下都不会"穿帮"
  美术只需要按真实物理调参数
  不用为每个场景重新"猜数值"
```

## 六、对比表格：Lambert → Phong → Blinn-Phong → PBR

| 对比维度 | Lambert | Phong | Blinn-Phong | PBR GGX |
|:--------|:-------|:-----|:-----------|:--------|
| **提出时间** | 1760 年（数学） | 1975 年 | 1977 年 | 2010 年代 |
| **漫反射** | ✅ 有 | ✅ 有 | ✅ 有 | ✅ 有 |
| **高光** | ❌ 没有 | ✅ 有 | ✅ 有（更快） | ✅ 有（物理正确） |
| **物理正确** | ❌ | ❌ | ❌ | ✅ 是 |
| **参数意义** | 抽象 | 抽象 | 抽象 | 对应真实物理 |
| **换环境需重调** | — | ✅ 需要 | ✅ 需要 | ❌ 不需要 |
| **性能开销** | ⭐ 最低 | ⭐⭐ 低 | ⭐ 最低 | ⭐⭐⭐ 中等 |
| **渲染效果** | ⭐ 石膏 | ⭐⭐⭐ 塑料 | ⭐⭐⭐ 塑料 | ⭐⭐⭐⭐⭐ 真实 |
| **使用年代** | 1990s 早期 | 1990s | **2000~2015** | **2015~至今** |

### 一个有趣的细节

```
Lambert 的公式是 1760 年提出的
距今已经 260 多年了

但今天的 PBR 渲染里
仍然包含 Lambert 的漫反射部分！

所以 Lambert 不是"被淘汰了"
而是"变成了 PBR 的一部分"
```

## 七、一句话总结

> **光照模型的进化就是一条"越来越像真的"路线：Lambert 只会算漫反射（像石膏），Phong 加上了高光（像塑料），Blinn-Phong 让高光算得更快（统治游戏界 20 年），最后 PBR GGX 用真正的物理定律来渲染（像真实世界）。每一步进化都让游戏画面离"真实"更近一步——从"看起来像"变成了"看起来就是"。**
>
> **所以下次你惊叹于某个游戏的画面时——记住，所有的光、所有的质感，都是一行行光照公式算出来的。只不过 2025 年的公式比 1995 年的公式，更懂物理了。** 💡
