# PBR 到底是个啥？—— 游戏画面从"塑料感"到"电影级"的秘密

> **灵魂拷问：你有没有觉得 10 年前的游戏画面有种"塑料玩具"的感觉？**
>
> 人物皮肤像蜡像、金属武器像喷了漆的塑料、整个场景怎么看怎么假。但现在的 3A 大作——比如《赛博朋克 2077》《最后生还者》——画面真实到让人分不清是游戏还是电影。
>
> 这中间最大的功臣，就是 **PBR（Physically Based Rendering，基于物理的渲染）**。

## 一、PBR 之前：游戏画面为什么"假"？

### 老式渲染：全靠"猜"和"凑"

在 PBR 出现之前（大概 2010 年以前），游戏画面的光照是靠**美术大佬手动调参数**凑出来的。

想象一下这个场景：

> 你要画一个金属水壶。在旧式渲染里，你需要自己"猜"：
> - 高光应该多亮？—— 调一个数字
> - 高光范围多大？—— 再调一个数字
> - 环境反射什么颜色？—— 还得调一个数字
>
> 调完之后，换个光照角度——完蛋，刚才的参数全废了，金属水壶看起来像个塑料壶。

这就是旧式渲染（Blinn-Phong 等经验模型）的核心问题：**参数是"经验值"，不是真实物理数据**。美术大佬只能靠肉眼反复试，一个材质调半天，换个环境又得重调。

### PBR 的革命：让物理定律来打工

PBR 的思路完全不一样：

> **别猜了，咱们直接模拟真实世界的光是怎么跑的。**

PBR 不是一种具体的技术，而是一套**渲染理念**——它遵循物理定律来计算光照，而不是靠经验公式"凑"出好看的效果。

它的核心原则只有三条：

## 二、核心一：能量守恒——光不会凭空产生也不会消失

### 物理课小复习

你可能忘了初中物理，但这条定律你一定听过：**能量既不会凭空产生，也不会凭空消失，只会从一种形式转化成另一种形式。**

光也是一种能量。所以在 PBR 里：

> **一个表面反射出去的光，绝对不能超过它接收到的光。**

### 用生活场景理解

🌊 想象海浪拍打礁石：

- 海浪的能量 = 100%
- 一部分能量被**反射**回去（海浪拍在礁石上溅起的水花）
- 剩下的能量被**吸收**（海水渗入礁石缝隙）
- 反射 + 吸收 = 100%，绝不会超过 100%

光打在物体表面也是一样的道理：

```
光照到表面：
      ↓
┌─────────────────┐
│  一部分反射出去  │  → 你看到的高光/镜面反射
│  一部分被吸收    │  → 变成热量，你看不见
│  一部分折射进去  │  → 在内部弹来弹去，再从别处出来（漫反射）
└─────────────────┘
      总和 = 100%
```

### 为什么这条规则这么重要？

因为旧式渲染**没有能量守恒**！美术可以调出一个参数，让反射光比入射光还亮——这在物理上是不可能的，但电脑不会拦着你。结果就是：**换个光照角度，画面就假得离谱。**

而 PBR 强制遵守能量守恒，所以**无论光照怎么变，画面的"真实感"都能保持住**。

## 三、核心二：金属 vs 非金属——反光方式完全不一样

### 一条分水岭

PBR 最核心的一个认知是：**金属和非金属的反光方式，从根本上就不一样。**

| 特性 | 金属（导体） | 非金属（绝缘体/电介质） |
|:----|:-----------|:-------------------|
| 反射光颜色 | **带自身颜色**（金反射黄光，铜反射橙光） | **纯白色**（反射光不带自身颜色） |
| 反射率 | **很高**（70%~100%） | **很低**（2%~5%） |
| 漫反射 | **几乎没有**（光被表面吸收） | **有**（光折射进去再出来） |
| 生活例子 | 金戒指、银叉子、铜锅 | 木头、塑料、石头、皮肤、布料 |

### 拿吃的来打比方

🍳 **非金属 = 鸡蛋羹**

- 表面有一层薄薄的**镜面反射**（蛋羹表面的光泽），反射光是**白色的**
- 大部分光穿进去，在内部弹来弹去，再出来——这就是你看到的**蛋羹本身颜色**

🥩 **金属 = 牛排刀**

- 光基本**穿不进去**，全在表面被反射
- 反射光**带刀的金属色**——银白色、钢灰色
- 所以金属看起来特别"亮"，而且反射的倒影特别清晰

### 为什么旧式渲染搞不定这个？

在旧式渲染里，金属和非金属用的是同一套参数——区别只是数值不同。这就导致：

> **旧式渲染的金属看起来总像"高光特别亮的塑料"。**

因为塑料（非金属）的反射光是白色的，你把它的高光调亮、范围调小，看起来像金属——但反射光还是白色的。而真实金属的反射光带自身颜色，这是本质区别。

PBR 用了一个巧妙的参数来区分：**金属度（Metalness）**。

- 金属度 = 0 → 非金属（木材、石头、皮肤）
- 金属度 = 1 → 金属（铁、金、铜）

有了这个参数，渲染器就知道该用哪种物理规则来计算光照了。

## 四、核心三：粗糙度——决定了高光是一个点还是一大片

### 微观世界的小山坡

再光滑的东西，在显微镜下都是坑坑洼洼的：

```
光滑表面（粗糙度低）：
_________________________
  \/\/\/\/\/\/\/\/\/\/\/
  微表面起伏很小

粗糙表面（粗糙度高）：
_________________________
  /\/\/\/\/\/\/\/\/\/\/\
  微表面起伏很大
```

### 用生活场景理解

💡 **手电筒照不同的墙：**

- **光滑的镜面**：手电筒的光会**整整齐齐地反射**到同一个方向，你看到一个清晰的亮点——这就是**锐利的高光**
- **粗糙的墙面**：每个微小凸起把光**反射到不同方向**，你看到的是一片**模糊的光斑**——这就是**模糊的高光**

### 粗糙度影响什么？

PBR 里的**粗糙度（Roughness）** 参数就是控制这个的：

| 粗糙度值 | 表面感觉 | 高光效果 | 生活例子 |
|:-------:|:--------|:--------|:--------|
| 0.0 ~ 0.1 | 镜面般光滑 | 清晰倒影，锐利高光 | 镜子、抛光金属、手机屏幕 |
| 0.2 ~ 0.4 | 半光滑 | 模糊倒影 | 不锈钢水槽、钢琴漆面 |
| 0.5 ~ 0.7 | 微粗糙 | 只有光晕，没倒影 | 磨砂塑料、未抛光的石头 |
| 0.8 ~ 1.0 | 非常粗糙 | 均匀散射，几乎没高光 | 粗布、水泥墙、树皮 |

### 金属度 + 粗糙度的黄金组合

这两个参数一起用，就能描述绝大多数真实材质：

| 材质 | 金属度 | 粗糙度 | 效果 |
|:----|:-----:|:------:|:-----|
| 黄金戒指 | 1.0 | 0.1 | 亮闪闪的金色反射 |
| 磨砂铁管 | 1.0 | 0.7 | 暗灰色，微弱的金属光泽 |
| 红色塑料 | 0.0 | 0.3 | 红色底色 + 白色高光 |
| 粗麻布 | 0.0 | 0.9 | 几乎纯漫反射，无光泽 |
| 陶瓷杯 | 0.0 | 0.2 | 光滑的白色表面，清晰高光 |

就这两个参数，加上底色，就能调出现实世界里 90% 的材质。这就是 PBR 的威力。

## 五、PBR 的"三件套"：Albedo、Normal、MR

游戏里最常用的 PBR 材质，通常由三张纹理贴图组成：

### ① Albedo（底色贴图）—— 物体"本来"的颜色

```
Albedo 贴图：
┌──────────────────────────┐
│                          │
│  记录物体"本身"的颜色     │
│  没有光照、没有阴影       │
│                          │
│  相当于：物体在白光下的   │
│          素颜照          │
└──────────────────────────┘
```

- **非金属**：Albedo 就是物体颜色（红砖墙 = 红褐色）
- **金属**：Albedo 记录的是反射颜色（金 = 金黄色）
- ⚠️ **Albedo 里不要带光照信息！** 这不是照片，是"素颜照"

### ② Normal（法线贴图）—— 让平面"看起来"凹凸不平

这是一个非常"骗人"但非常高效的技术：

> 一个平坦的墙面，只有 2 个三角形——但贴上法线贴图后，看起来像有几百个凹凸不平的砖块。

原理很简单：

```
实际几何：
  ────────────────  只有 2 个三角形，平得跟纸一样
  
法线贴图"骗"GPU：
  每个像素告诉你"我这个位置应该朝哪个方向反光"
  ── 砖缝处朝侧面反光 → 看起来像凹进去了
  ── 砖面处朝正面反光 → 看起来像凸出来了
  
结果：眼睛被"骗"了，以为真的有立体凹凸
```

### ③ MR（Metallic + Roughness）—— 一张图存两个参数

为了省空间，PBR 把金属度和粗糙度存在**同一张贴图**的不同颜色通道里：

```
MR 贴图：
┌──────────────────────────┐
│ R 通道 → 粗糙度           │
│ G 通道 → 金属度           │
│ B 通道 → 空着（或环境光遮蔽）│
└──────────────────────────┘
```

| 通道 | 存什么 | 值 = 0 表示 | 值 = 1 表示 |
|:----:|:------|:----------|:----------|
| R | 粗糙度（Roughness） | 镜面光滑 | 非常粗糙 |
| G | 金属度（Metallic） | 非金属 | 纯金属 |
| B | AO（环境光遮蔽，可选） | 无遮挡 | 完全遮挡 |

### 三张图一起用的效果

```
Albedo（素颜照）  +  Normal（凹凸假象）  +  MR（物理参数）
      ↓                    ↓                    ↓
  物体的颜色          表面的凹凸感          怎么反光
      └────────────────┬───────────────────┘
                       ↓
                最终渲染效果
            = 真实到可怕的材质
```

## 六、PBR 之前 vs PBR 之后：对比表格

| 对比维度 | 旧式渲染（Blinn-Phong） | PBR（基于物理） |
|:--------|:---------------------|:--------------|
| **核心思路** | 经验公式，"看起来像就行" | 物理模拟，"真实世界怎么跑" |
| **能量守恒** | ❌ 不保证 | ✅ 强制遵守 |
| **金属处理** | 调高光参数"假装"是金属 | 用 Metalness 区分，物理正确 |
| **参数意义** | 抽象数值，靠肉眼调 | 对应真实物理属性 |
| **换光照环境** | 得重新调参数 | 不用动，物理自动适配 |
| **美术工作流** | 每个环境都要调，重复劳动 | 一次调好，到处能用 |
| **真实感上限** | ⭐⭐⭐ 塑料感明显 | ⭐⭐⭐⭐⭐ 电影级真实 |
| **性能开销** | 低 | 略高（但现代显卡完全扛得住） |

### 为什么 PBR 成了业界标配？

2015 年以后，**几乎所有 3A 游戏和商业引擎都转向了 PBR**：

- **Unity**：2014 年加入 Standard Shader（PBR）
- **Unreal Engine**：从诞生就基于 PBR 设计
- **Frostbite（寒霜）**、**Snowdrop（雪花莲）**：全部 PBR
- **DSEngine**：同样基于 PBR 渲染

原因很简单：**PBR 让美术和程序都省心了**。美术只需要按照真实世界的物理属性调参数，不用再为每个场景"猜"数值。而引擎这边，物理定律是普适的——光照怎么变，画面都不会"穿帮"。

---

## 七、一句话总结

> **PBR 不是什么黑科技，它就是让游戏渲染"遵守物理定律"——能量守恒、金属与非金属分开算、粗糙度决定高光形状。有了这三条规则，游戏画面终于从"塑料玩具"进化到了"电影级真实"。**
>
> **以后你打游戏的时候可以留意一下：那个金属水壶上清晰的倒影、那块木头柔和的高光——背后都是 PBR 在默默打工。** 😎
