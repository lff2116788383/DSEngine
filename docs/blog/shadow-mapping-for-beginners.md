## 一、影子本质：光源看不到的地方就是阴影

### 一个再简单不过的道理

> **阴影的本质就是：光源发出的光被某样东西挡住了，照不到后面的区域。**

你站在太阳底下，太阳光照到你身上——你的身体挡住了光，地上就多了一块暗区，这就是影子。

### 用生活场景理解

🔦 **手电筒照墙：**

```
你拿着手电筒：
                 手电筒（光源）
                    │
                    │ 光线
                    │
               ┌────┴────┐
               │  你的手  │  ← 挡住了光
               └────┬────┘
                    │
              ┌─────┴─────┐
              │  墙上暗区  │  ← 阴影
              └───────────┘
```

阴影的形成只需要三个条件：
1. **光源**（手电筒/太阳/灯泡）
2. **遮挡物**（你的手/角色/建筑）
3. **接收面**（墙壁/地面/角色自己）

### 但问题是——电脑怎么知道哪里被挡住了？

现实世界中，光线一直在跑，眼睛看到的就是结果。但在游戏里，我们需要**告诉电脑"哪里被挡住了"**。最常用的方法就是——**Shadow Mapping（阴影贴图）**。

## 二、Shadow Mapping：从光源拍一张"深度照片"

### 核心思路

Shadow Mapping 的思路非常巧妙：

> **从光源的位置"拍一张照片"，记录每个方向上的东西离光源有多远。然后渲染场景时，检查每个像素是不是比这张"照片"里的距离更远——更远就说明被挡住了。**

### 两步走

```
Step 1：从光源视角渲染"深度图"
  
  把相机放到光源的位置，朝场景拍一张照片
  但这张照片不记录颜色，只记录"每个像素离光源多远"
  这张照片就叫 Shadow Map（阴影贴图）
  
Step 2：从主相机视角渲染场景
  
  对每个像素：
    ① 算算这个像素在光源那个"照片"里的位置
    ② 比较两个距离：
       这个像素离光源的距离 VS 光源"照片"里记录的距离
    ③ 如果像素的距离 > 照片里的距离 → 说明被挡住了 → 画成阴影
    ④ 如果像素的距离 = 照片里的距离 → 说明没被挡住 → 正常画
```

### 用生活场景理解

📸 **排队拍照的梗：**

想象你在游乐园排队，前面有一个人比你高：

```
光源 = 站在高台上的摄影师
你 = 被挡住的那个人

Step 1：
  摄影师（光源）拍了一张队伍的照片
  照片里只记录了"每个人离摄影师多远"
  前排的高个子记录为"2 米"
  
Step 2：
  现在要画你（某个像素）
  先查一下你在光源照片里的位置
  你离光源其实只有 2.1 米
  但照片里记录那个位置被高个子占了（2 米）
  2.1 > 2 → 你被挡住了！
  结论：你处于阴影中
```

### 伪代码

```
// Step 1：从光源渲染深度图
function renderShadowMap():
    camera.position = light.position    // 把相机放到光源位置
    camera.direction = light.direction  // 朝光源方向看
    for each pixel in shadowMap:
        shadowMap[pixel] = depthValue  // 只存深度，不存颜色

// Step 2：主渲染时判断阴影
function isInShadow(pixelWorldPos, light):
    // 把像素位置转换到光源的视角
    lightSpacePos = convertToLightSpace(pixelWorldPos)
    
    // 查 Shadow Map 里这个位置记录的深度
    recordedDepth = shadowMap[lightSpacePos.xy]
    
    // 当前像素离光源的距离
    myDepth = lightSpacePos.z
    
    // 如果我的深度 > 记录的深度 → 被挡住了
    return myDepth > recordedDepth
```

### 这个方案有个小问题

直接比较深度会有**精度误差**。因为 Shadow Map 是一张纹理，它存的是离散的数值，而场景是连续的。这就会导致一些问题——

## 三、阴影的质量问题：锯齿和飘动

### 问题 1：阴影锯齿（Shadow Aliasing）

Shadow Map 本身是有分辨率的——比如 1024×1024。想象一下：

```
Shadow Map 分辨率不够时：
  一个像素在 Shadow Map 里对应场景中一大块区域
  这一大块区域要么全亮要么全暗
  结果：阴影边缘出现明显的"马赛克"锯齿
```

**解决办法：**
- 提高 Shadow Map 分辨率（更耗性能）
- 用**PCF（Percentage Closer Filtering）**——在阴影边缘采样多个点然后平均，让边缘变模糊、变平滑

### 问题 2：阴影痤疮（Shadow Acne）

这个名字很形象——阴影表面像长了痘痘一样，出现奇怪的花纹。

**原因：** 深度比较时的精度误差。比如有一段斜坡：

```
真实情况：
  斜坡表面 → 离光源距离从 2.000 渐变到 2.100
  
Shadow Map 精度不够：
  可能记录为 2.000 ~ 2.005 之间的一个值
  斜坡上有些点比 Shadow Map 深一点点
  结果：这些点被误判为"被挡住了"
  画面出现斑驳的"痘痘"
```

**解决办法：** 加一个**偏差值（Bias）**——比较时故意让 Shadow Map 的值偏小一点，给一个"容错空间"。但 Bias 加多了，阴影会"飘"起来——这叫 **Peter Panning（彼得潘现象）**，就是阴影和物体之间出现缝隙，像小飞侠彼得潘一样飘在空中。

### 问题 3：Peter Panning（阴影飘动）

```
Bias 太大：
  阴影离开物体底部，悬空了
  看起来像物体浮在半空中
  
Bias 太小：
  阴影痤疮又回来了
```

这就是阴影渲染里最经典的**权衡问题**：Bias 调来调去，总是顾此失彼。

## 四、CSM：近处高清、远处低清——分成三层

### 问题的起源

Shadow Map 的分辨率是固定的（比如 4096×4096）。但场景很大——近处的物体可能只占 Shadow Map 的一小部分，远处却占了一大片。

```
传统 Shadow Map：
  ┌──────────────────────────────┐
  │                              │
  │  近处的一棵树      远处的山  │
  │  （只占 10 像素）   （占 90%）│
  │                              │
  │  结果：近处的树影全是锯齿    │
  │        远处的山影倒是挺清晰  │
  └──────────────────────────────┘
  
  但人眼盯着近处看，锯齿特别明显！
```

### CSM 的思路

**CSM（Cascaded Shadow Maps，级联阴影贴图）** 的解决方案非常聪明：

> **不要用一张 Shadow Map 覆盖整个场景，而是分成几层——近处用高分辨率，远处用低分辨率。**

```
CSM 分层示意（俯视图）：
  
  ┌──────────────────────────────────┐
  │          Level 3（远处）         │
  │    ┌──────────────────────┐      │
  │    │   Level 2（中距离）  │      │
  │    │   ┌──────────────┐  │      │
  │    │   │ Level 1      │  │      │
  │    │   │ （近处）      │  │      │
  │    │   │              │  │      │
  │    │   └──────────────┘  │      │
  │    └──────────────────────┘      │
  │     相机                          │
  └──────────────────────────────────┘
```

### 三级 CSM 的分配

| 层级 | 覆盖距离 | Shadow Map 分辨率 | 像素在屏幕上的大小 |
|:----:|:--------|:----------------:|:-----------------|
| Level 1 | 0~20 米 | 2048×2048 | 很大（近处细节高清） |
| Level 2 | 20~60 米 | 2048×2048 | 中等 |
| Level 3 | 60~150 米 | 2048×2048 | 很小（远处模糊也没事） |

这样，近处用同样分辨率的高清 Shadow Map，远处即使模糊了也没人注意。

### 层级之间的过渡

CSM 有个小问题：从 Level 1 到 Level 2 的边界上，阴影的分辨率突然变化——会出现一条**明显的分界线**。

**解决方法：** 在边界处做**渐变过渡（Blend）**——在过渡区域同时采样两个层级的 Shadow Map，然后按距离混合，让边界看不出来。

## 五、PCSS：阴天和晴天的阴影为什么不一样软？

### 生活中的观察

你有没有注意到：

| 天气 | 阴影特点 | 原因 |
|:----|:--------|:-----|
| ☀️ 晴天正午 | 阴影边缘**锐利、清晰** | 太阳是一个小光源，光线几乎平行 |
| ☁️ 阴天 | 阴影边缘**模糊、柔和** | 云层把光散射了，光源变成了一个大面积光源 |
| 🌅 黄昏 | 阴影**越来越长、越来越软** | 太阳角度低，光线经过更多大气层 |

### 核心原理

阴影的"软硬"取决于**光源的大小**：

```
点光源（小光源）→ 锐利阴影：
  光源 ───────→ ⬛ 遮挡物 → ███ 清晰阴影

面光源（大光源）→ 软阴影：
  光源┌──┐    → ⬛ 遮挡物
      └──┘       ├── 完全阴影（本影）
                 └── 半阴影（半影）← 边缘模糊
```

### PCSS 是什么？

**PCSS（Percentage Closer Soft Shadows，百分比渐近软阴影）** 是一种能生成软阴影的技术。

它的思路也很巧妙：

> **阴影有多软？取决于遮挡物离光源有多远、离接收面有多远。**

```
PCSS 三步走：

Step 1：找遮挡物
  对每个像素，在 Shadow Map 里找它附近有没有遮挡物
  算平均遮挡深度

Step 2：算半影大小
  遮挡物离光源越近 → 半影越大 → 阴影越软
  遮挡物离接收面越远 → 半影越大 → 阴影越软
  
  公式直观理解：
    半影大小 = (接收面距离 - 遮挡物距离) / 遮挡物距离
    这个值越大 → 阴影越模糊

Step 3：自适应滤波
  根据算出来的半影大小，决定采样范围
  半影大 → 多采样几个点 → 模糊一点
  半影小 → 少采样几个点 → 锐利一点
```

### 用生活场景理解

🎥 **电影幕布投影：**

- **投影仪离幕布近** → 你的手靠近投影仪，幕布上的手影**边缘模糊**
- **投影仪离幕布远** → 同样的手，幕布上的影子**边缘清晰**
- 如果投影仪本身是个小灯泡（小光源）→ 影子清晰
- 如果换成日光灯管（大光源）→ 影子模糊

PCSS 就是在模拟这个物理现象——根据场景的几何关系，自动决定阴影应该多软。

## 六、Contact Shadow：物体接触地面的精细阴影

### 还有一个细节

CSM 和 PCSS 解决了大范围的阴影问题，但有一个地方它们处理不好：

> **物体和地面接触的地方——比如人物的脚底、桌腿和地板接触的位置。**

因为 Shadow Map 的分辨率有限，这些"接触点"的阴影要么锯齿严重，要么干脆就是一条线。

### Contact Shadow 的解决方案

**Contact Shadow（接触阴影）** 是一个专门的补充技术：

> **只在物体和表面的接触位置做精细的阴影计算，补充普通 Shadow Map 丢失的细节。**

```
没有 Contact Shadow：        有 Contact Shadow：
  ┌────────────┐             ┌────────────┐
  │  物体      │             │  物体      │
  │  ██████    │             │  ██████    │
  │  ──────    │  ← 接触处  │  ██████    │
  │  地面      │  阴影模糊  │  ■■■■■■    │  ← 接触处有精细阴影
  └────────────┘             └────────────┘
```

它的原理其实是一种**屏幕空间技术**——在最终画面上沿着物体边缘往地面方向采样，在地面附近增加一条精细的暗线，模拟接触阴影。

## 七、对比表格：各种阴影技术

| 技术 | 解决的问题 | 优点 | 缺点 | 性能开销 |
|:----|:----------|:----|:----|:--------:|
| **Shadow Map** | 基础阴影 | 实现简单，通用 | 锯齿，精度问题 | 低 |
| **PCF** | 阴影锯齿 | 边缘平滑，效果好 | 需要多次采样 | 中 |
| **CSM** | 近处锯齿远处浪费 | 近处高清远处省 | 层级边界有过渡问题，多张 Shadow Map 占内存 | 中高 |
| **PCSS** | 软阴影 | 物理正确，真实感强 | 计算量大，需要多次采样 | 高 |
| **Contact Shadow** | 接触细节 | 补充精细阴影 | 只在屏幕空间有效，可能漏掉 | 低 |
| **VSM（方差阴影贴图）** | Shadow Map 锯齿 | 可以模糊，不用多次采样 | 有漏光问题 | 中 |

### 现实中的组合

现代游戏通常不会只用一种技术，而是**组合拳**：

```
基础框架：CSM（处理远近阴影）
  + PCF（平滑边缘）
  + PCSS（可选，处理软阴影）
  + Contact Shadow（补充接触细节）
  
最终效果：远处模糊、近处清晰、边缘平滑、接触处精细
```

## 八、一句话总结

> **游戏里的影子，说白了就是从光源位置拍一张"深度照片"（Shadow Map），渲染每个像素时查一下这张照片——如果它比照片记录的距离远，说明被挡住了，就是阴影。为了做得更好看，CSM 把远近分层处理，PCSS 让阴影该软就软、该硬就硬，Contact Shadow 补充接触处的精细阴影。**
>
> **下次玩游戏看到夕阳下长长的软影子，你可以跟朋友说："看，这个 PCSS 调得不错"——然后深藏功与名。** 🌇
