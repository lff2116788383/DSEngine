## 一、CPU vs GPU：博士生 vs 一万个小学生

### 先搞明白 CPU 和 GPU 的区别

要理解 Shader，得先搞明白 CPU 和 GPU 是什么关系。

看这张对比表：

| 对比项 | CPU（中央处理器） | GPU（图形处理器） |
|:------|:---------------|:---------------|
| **擅长** | 处理复杂、多样的任务 | 处理大量、重复的任务 |
| **核心数** | 6~16 个（家用电脑） | 几千个（显卡里） |
| **单个核心能力** | 超级强（博士生） | 很弱（小学生） |
| **工作方式** | 串行——一个一个来 | 并行——大家一起上 |
| **比喻** | 一个十项全能的博士 | 一万个只会加减法的小学生 |

### 用生活场景理解

🏪 **开超市理货：**

**CPU 模式：** 只有一个超级能干的店长。
- 他一个人搞定进货、摆货、收银、打扫卫生
- 每一件事都做得很好，但一次只能做一件
- 遇到新问题，他能随机应变

**GPU 模式：** 一万个只做一件事的小学生。
- 每个小学生只会摆一箱货
- 但一万人同时上手，一万箱货瞬间摆完
- 遇到新问题？他们不会——需要你提前写好"操作说明"

### 所以 Shader 就是"操作说明"

回到游戏：你要在屏幕上画一个 3D 模型。

- 这个模型可能有 **10 万个顶点、几百万个像素**
- CPU 来处理？串行算完，黄花菜都凉了
- 所以交给 GPU——一万个小学生同时开工

但 GPU 的小学生**不会思考**，他们需要你提前写好：

> **"每个顶点该怎么处理？"** → 写一个 *Vertex Shader*
> **"每个像素该怎么上色？"** → 写一个 *Fragment Shader*

这两份"操作说明"合起来，就是最核心的**着色器（Shader）**程序。

## 二、什么是"着色器"？—— 给 GPU 的"怎么画"指令

### 正式定义

> **着色器（Shader）是一段运行在 GPU 上的小程序，它告诉 GPU 如何渲染每个顶点和每个像素。**

### 通俗理解

你去吃自助烤肉：

```
你（程序员） → 写下规则："牛肉烤 3 分钟翻面，鸡翅烤 5 分钟"
               ↓
      这是 Shader（规则说明书）
               ↓
烤肉师傅（GPU） → 严格按照规则执行
               ↓
      成百上千块肉同时烤（并行处理）
               ↓
      最终：所有肉同时上桌（画面渲染完成）
```

### Shader 的几个特点

1. **运行在 GPU 上**，不是 CPU——所以速度极快
2. **并行执行**——同一个 Shader 同时运行在成百上千个顶点/像素上
3. **典型的 Shader 只有几十行代码**——但被执行了百万次
4. **不同阶段有不同的 Shader**——各司其职

### 动画理解

想象一下：

```
模型文件（.obj/.fbx）：
  ┌─ 顶点数据（位置、法线、UV） ─┐
  │  顶点 1: (0,0,0)             │
  │  顶点 2: (1,0,0)             │
  │  顶点 3: (0,1,0)             │
  │  ... 共 10 万个顶点          │
  └─────────────────────────────┘
                   ↓
      Vertex Shader（处理每个顶点）
                   ↓
      光栅化（把三角形变成像素）
                   ↓
      Fragment Shader（处理每个像素）
                   ↓
      最终画面输出到屏幕
```

## 三、Vertex Shader：每个顶点怎么动

### 它干什么？

> **Vertex Shader 处理的是"顶点"——也就是 3D 模型里那些"拐点"。**

一个 3D 模型由成千上万个三角形组成，每个三角形有三个顶点。Vertex Shader 的任务就是：**告诉 GPU 每个顶点应该放在屏幕的什么位置。**

### 用生活场景理解

🎨 **你是一个雕塑家：**

- 你手里有一个铁丝网做的人形骨架（这就是"顶点数据"）
- Vertex Shader 就是你给骨架做的"变形指令"：
  - "把右臂抬起来 30 度" → 肩关节的顶点位置变了
  - "把膝盖弯曲 90 度" → 膝关节的顶点位置变了
  - "整个人放大 2 倍" → 所有顶点位置都变了
- 骨架变形好了，下一步才是往上糊泥巴（上色）

### Vertex Shader 具体做什么？

**三件核心工作：**

```
① 坐标变换：把 3D 世界坐标 → 屏幕 2D 坐标
   
   模型坐标 → 世界坐标 → 相机坐标 → 屏幕坐标
   （把模型放到场景里 → 用相机拍 → 映射到屏幕上）

② 顶点变形：让模型动起来
    
    顶点蒙皮（骨骼动画）：肩膀动 → 整个手臂的顶点跟着动
    顶点位移：水面波动、旗帜飘动

③ 传递数据：把需要的信息告诉 Fragment Shader
    
    "这个顶点的颜色信息传下去"
    "这个顶点的纹理坐标传下去"
    "这个顶点的法线方向传下去"
```

### 伪代码感受一下

```
// Vertex Shader 的"思维逻辑"
function vertexShader(vertexInput):
    // 1. 把顶点从 3D 位置变成屏幕 2D 位置
    screenPos = projectToScreen(vertexInput.position)
    
    // 2. 如果有骨骼动画，计算顶点偏移
    if hasAnimation:
        screenPos = applySkinning(screenPos, boneData)
    
    // 3. 把需要的信息传给 Fragment Shader
    passToFragment({
        uv: vertexInput.uv,             // 纹理坐标
        normal: vertexInput.normal,     // 法线方向
        worldPos: screenPos             // 世界坐标
    })
    
    return screenPos
```

## 四、Fragment Shader：每个像素什么颜色

### 它干什么？

> **光栅化之后——也就是 GPU 知道"哪些像素在三角形内部"之后——就该 Fragment Shader 上场了。**

它的任务很简单但也很复杂：**决定每个像素最终显示什么颜色。**

### 用生活场景理解

🖍️ **你有一本填色书：**

- 书的轮廓已经画好了（Vertex Shader 完成了顶点定位）
- 现在每个区域（像素）需要填什么颜色？
- Fragment Shader 就是"填色说明书"：
  - "这个区域涂红色" → 纯色
  - "这个区域按照纹理图案涂" → 贴纹理
  - "这个区域靠近灯光，涂亮一点" → 光照计算
  - "这个区域在阴影里，涂暗一点" → 阴影处理

### Fragment Shader 具体做什么？

**每像素的计算：**

```
① 纹理采样：从贴图上取颜色
    
    模型告诉 GPU "这个像素对应纹理的 (u,v) 位置"
    GPU 从纹理图片上取出那个位置的颜色

② 光照计算：根据光源信息算颜色
    
    灯光方向、表面朝向（法线）、观察方向
    三者决定了这个像素看起来是亮还是暗
    这就是 PBR 光照计算发生的地方

③ 阴影处理：判断像素是否在阴影里
    
    如果在阴影里 → 颜色变暗
    如果在半影区 → 颜色过渡

④ 混合：把多层效果叠在一起
    
    基础颜色 + 光照 + 阴影 + 雾效 + 后处理
    最后输出一个最终颜色
```

### 伪代码感受一下

```
// Fragment Shader 的"思维逻辑"
function fragmentShader(fragmentInput):
    // 1. 从纹理上取颜色
    baseColor = sampleTexture(fragmentInput.uv)
    
    // 2. 计算光照
    lightDir = normalize(lightPosition - fragmentInput.worldPos)
    normal = fragmentInput.normal
    
    // 3. 漫反射：光越直射，越亮
    diffuse = max(0, dot(normal, lightDir)) * lightColor
    
    // 4. 高光：镜面反射
    viewDir = normalize(cameraPosition - fragmentInput.worldPos)
    specular = computeSpecular(normal, lightDir, viewDir, roughness)
    
    // 5. 最终颜色 = 基础色 × 漫反射 + 高光
    finalColor = baseColor * diffuse + specular
    
    // 6. 如果是阴影，变暗
    if isInShadow:
        finalColor *= 0.3
    
    return finalColor
```

## 五、渲染管线的完整流水图

把前面讲的串起来，就是完整的渲染管线：

```
┌─────────────────────────────────────────────────────────────┐
│                    完整渲染管线流水线                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ① 应用阶段（CPU）                                           │
│     └─ 准备场景数据、做剔除、发 Draw Call                    │
│                                                             │
│  ② Vertex Shader（GPU）                                      │
│     └─ 处理每个顶点：坐标变换 + 变形 + 传数据                │
│                                                             │
│  ③ 光栅化（GPU）                                             │
│     └─ 把三角形"打碎"成像素，决定哪些像素被覆盖              │
│                                                             │
│  ④ Fragment Shader（GPU）                                    │
│     └─ 处理每个像素：纹理采样 + 光照 + 阴影 + 混合          │
│                                                             │
│  ⑤ 输出合并（GPU）                                           │
│     └─ 深度测试、透明度混合、输出到屏幕                      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 类比：盖房子

| 阶段 | 盖房子 | Shader 阶段 |
|:----|:------|:-----------|
| 设计图纸 | 准备模型数据 | 应用阶段 |
| 搭钢筋骨架 | 确定每个顶点的位置 | Vertex Shader |
| 砌砖填缝 | 把三角形变成像素 | 光栅化 |
| 粉刷墙面 | 给每个像素上色 | Fragment Shader |
| 验收交付 | 最终输出到屏幕 | 输出合并 |

## 六、其他类型的着色器

除了上面两个最核心的，还有几种常见的 Shader：

### Geometry Shader（几何着色器）—— "造物主"

- **能做什么？** 根据已有的顶点，**创造出新的顶点**
- **用途举例：** 把一个点变成一个草叶、把一个线段拉成一片草地
- **现状：** 比较老了，现代引擎用得少

### Compute Shader（计算着色器）—— "万能工具"

- **能做什么？** 不干渲染的事，而是用 GPU 做通用计算
- **用途举例：** 物理模拟、粒子系统、后处理特效
- **优势：** 可以充分利用 GPU 的并行计算能力，干 CPU 干得慢的活

### Tessellation Shader（曲面细分着色器）—— "细节狂魔"

- **能做什么？** 把粗糙的模型细化，增加更多三角形
- **用途举例：** 地面从远处看是平的，走近了自动出现凹凸细节
- **省性能的妙招：** 远处低模、近处细分——不比加载精细模型慢

### 各 Shader 在管线中的位置

```
Vertex Shader → Tessellation Shader → Geometry Shader → Fragment Shader
                                                              ↑
                                                       Compute Shader
                                                     （独立，任意位置调用）
```

## 七、一句话总结

> **着色器（Shader）就是写给 GPU 的"操作说明书"——Vertex Shader 告诉 GPU 每个顶点放在哪，Fragment Shader 告诉 GPU 每个像素涂什么颜色。没有 Shader，显卡就像一群只会站着发呆的小学生，有了 Shader，他们才能整齐划一地画出你看到的每一帧游戏画面。**
>
> **下次打游戏卡顿的时候，别只骂显卡了——也许是你家的 Shader 写得太费了呢！** 😏
