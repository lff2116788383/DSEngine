## 三、商业模式设计

### 3.1 分层定价模型

```
层级             定价              核心权益
────────────────────────────────────────────────
免费版 (Community)   ¥0
  ├── 引擎完整源码（MIT 或自定义开源许可）
  ├── 基础编辑器
  ├── Lua 脚本系统
  ├── 发布到 Windows（带 DSE 启动画面）
  └── 社区论坛支持

创作者版 (Creator)   ¥599/年   或 ¥1,999 永久
  ├── 全部免费版功能
  ├── 去除启动画面
  ├── 命令行打包工具（含加密选项）
  ├── 优先技术支持（24h 响应）
  └── 官方教程体系全部解锁

专业版 (Pro)         ¥2,999/年
  ├── 全部创作者版功能
  ├── 商业授权（无分成，无版税）
  ├── 私有插件/模块支持
  ├── 源码级技术支持（12h 响应）
  ├── 定制引擎组件
  └── SDK 优先体验权

企业版 (Enterprise)   定制报价
  ├── 全部专业版功能
  ├── 源代码完全所有（可闭源修改）
  ├── 专属技术支持群（2h 响应）
  ├── 引擎工程师驻场支持（可选）
  ├── 定制功能开发（按人天计费）
  └── 培训服务
```

### 3.2 定价策略说明

**为什么有永久的创作者版？**
- 独立开发者最反感"订阅制"（Unity 的教训）
- 永久授权作为信任锚点：用户买得放心
- 年费版提供持续收入，永久版提供初始现金流

**为什么免费版不带加密且去不掉启动画面？**
- 加密本身防不了破解，但可以作为付费分界线
- 启动画面是免费的"广告位"——每个用 DSE 做的免费游戏都是一次宣传

**为什么不收版税（Royalty）？**
- 市场证明版税模式严重伤害开发者信任（Unity Runtime Fee 的教训）
- 只有垄断地位的引擎（UE）才能收版税
- DSE 在成长期需要的是**用户规模**，不是单用户收入

## 七、风险与应对

### 7.1 风险矩阵

| 风险 | 概率 | 影响 | 应对策略 |
|:-----|:----:|:----:|:---------|
| **没人用** | 🟡 中 | 🔴 致命 | 先聚焦"学习者/学生"群体，降低预期。引擎学习社区本身就有刚需 |
| **Unity/Godot 更新狙击** | 🟢 低 | 🟡 中 | DSE 的生态位不在"和它们抢用户"，在"不同的赛道"（可阅读/教学/轻量） |
| **作者弃坑** | 🟡 中 | 🔴 致命 | 保持持续输出，不要停更超过 3 个月。GitHub 上引擎项目最怕的不是不好用，是"死了" |
| **代码被盗用/闭源** | 🟢 低 | 🟢 低 | 核心引擎代码不开源或选择许可证。基于 DSE 做商业游戏需要授权 |
| **法律风险（PhysX/NVIDIA）** | 🟢 低 | 🟡 中 | PhysX 是可选能力，默认关闭。避免核心功能依赖第三方闭源库 |
| **个人/生活变动** | 🟡 中 | 🟡 中 | 文档和架构设计必须足够好，以便中断 3-6 个月后还能重新上手 |

### 7.2 风险应对核心原则

> **DSE 的风险管理只有一条：不要停。** 技术可以不好，功能可以不全，但绝对不能看起来像"废弃项目"。Godot 用了 7 年内部开发 + 4 年极低收入期才活过来。DSE 需要同样的耐心。

## 九、竞品差异化策略

### 9.1 定位声明

> **DSEngine 的目标不是做一个"更好的 Unity/UE/Godot"，而是做一个人可以读懂的、架构现代的、开箱可用的游戏引擎。**

### 9.2 与竞品的差异化

| 对比对象 | DSE 的优势 | 应对质疑的说辞 |
|:---------|:-----------|:---------------|
| **vs Unity** | 无 Runtime Fee、包体小(~15MB)、源码完全可控 | "Unity 很好，但如果你想要一个你可以完全掌控的、没有政策风险的引擎，DSE 是选项" |
| **vs Unreal** | 12 万行可读代码、无版税、启动快 | "UE5 很强，但你能读完它全部源码吗？DSE 可以。" |
| **vs Godot** | 纯正 ECS 架构、PhysX 物理、3D 模块更丰富、三端渲染 | "Godot 是开源界的 Unity，DSE 是开源界的一个新物种——ECS 驱动、PhysX 原生、3D 能力更强" |
| **vs B 站自研引擎** | 完成度（~12 万行 vs ~1-5 万行）、文档体系、持续更新 | 不需要对比——不在一个量级 |

### 9.3 核心卖点提炼（一句话营销）

| 场景 | 一句话 |
|:-----|:-------|
| **对学生** | "如果你想理解游戏引擎是怎么工作的，读 DSE 源码比读 UE 源码现实 100 倍" |
| **对独立开发者** | "11.8 万行代码，PBR + TAA + 三后端，全部掌控在你手里" |
| **对工作室** | "ECS + PhysX + DDGI，2 周内评估你的项目是否能跑在 DSE 上" |
| **对技术决策者** | "没有 Runtime Fee，没有版税，没有 Vendor Lock-in" |

## 十一、总结

> **DSEngine 的商业闭环不需要融资、不需要团队、不需要爆款。它只需要两样东西：持续的代码产出和稳定的社区运营。**
