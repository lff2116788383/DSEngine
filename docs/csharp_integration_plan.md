## 2. 技术选型：.NET 8 CoreCLR (hostfxr)

### 为什么不选 Mono

| 维度 | Mono Embedded | .NET 8 CoreCLR | 结论 |
|---|---|---|---|
| 维护状态 | 进入 .NET 统一后的维护模式 | 微软官方主线 | CoreCLR ✓ |
| JIT 性能 | 一般 | RyuJIT, 显著更快 | CoreCLR ✓ |
| 发布选项 | AOT (Mono) | NativeAOT (零运行时) | CoreCLR ✓ |
| 迁移成本 | 先 Mono 后 CoreCLR = 两次改模板 | 一步到位 | CoreCLR ✓ |
| InternalCall | 需逐条 `mono_add_internal_call` | `[LibraryImport]` 零注册 | CoreCLR ✓ |
| 热重载 | AppDomain (已废弃) | `AssemblyLoadContext` | CoreCLR ✓ |
| 反向调用 C→C# | delegate marshal | `delegate* unmanaged` 函数指针 | CoreCLR ✓ |
| 缺点 | — | 需 .NET SDK 或 self-contained 发布 | 可接受 |

## 3. 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│  C# User Scripts (GameScripts/Game/*.cs)                     │
│    class PlayerController : DseScript { ... }                │
├─────────────────────────────────────────────────────────────┤
│  C# Engine Binding Layer (generated + handwritten)           │
│    Native.gen.cs — [LibraryImport("dse_engine")] 声明        │
│    高级封装: Entity, Transform, MeshRenderer, Vector3, ...   │
├─────────────────────────────────────────────────────────────┤
│  .NET 8 Runtime Host (engine/scripting/csharp/)              │
│    CSharpHost.h/cpp — hostfxr 初始化, ALC 热重载,            │
│    反向回调注册 (C→C# delegate* unmanaged)                   │
├─────────────────────────────────────────────────────────────┤
│  C ABI (dse_api.h) ← 已 100% 完成                           │
├─────────────────────────────────────────────────────────────┤
│  Engine Core (ECS / Render / Physics / Audio)                │
└─────────────────────────────────────────────────────────────┘
```

### 调用方向

```
C# → C:    [LibraryImport("dse_engine")] → dse_api.h 导出符号 (P/Invoke, 无注册)
C → C#:    engine 调用 hostfxr 获取函数指针 → managed delegate (生命周期/事件回调)
```

## 5. 实施阶段

### Phase 1: 核心接入 (目标: C# 脚本能调引擎 API)

#### 1.1 更新 Codegen 模板 — `csharp_native.cs.j2`

从 Mono InternalCall 改为 .NET 8 `[LibraryImport]`:

```csharp
// <auto-generated> 来源：binding_defs.json — 勿手动修改
using System.Runtime.InteropServices;

namespace DSEngine;

internal static partial class Native {
    const string Lib = "dse_engine";

    // ---- TransformComponent ----
    [LibraryImport(Lib, EntryPoint = "dse_transform_get_position")]
    internal static partial void dse_transform_get_position(uint e, out float x, out float y, out float z);
    
    [LibraryImport(Lib, EntryPoint = "dse_transform_set_position")]
    internal static partial void dse_transform_set_position(uint e, float x, float y, float z);
    // ...
}
```

优势: **零注册代码** — .NET source generator 在编译时生成 marshal stub。

#### 1.2 创建 C# 项目结构


#### 1.3 实现 CSharpHost (C++ 侧)

```cpp
// engine/scripting/csharp/csharp_host.h
#pragma once
#include <string>
#include <functional>

// 前向声明 hostfxr 类型
typedef void* hostfxr_handle;

class CSharpHost {
public:
    bool initialize(const std::string& runtime_config_path);
    void shutdown();
    bool reload();  // AssemblyLoadContext 卸载→重载
    
    // 生命周期 (C → C# 方向)
    void invoke_start();
    void invoke_update(float dt);
    void invoke_fixed_update(float dt);
    
    bool is_loaded() const { return host_handle_ != nullptr; }

private:
    hostfxr_handle host_handle_ = nullptr;
    
    // 从 managed 侧获取的函数指针
    using fn_void    = void(*)();
    using fn_float   = void(*)(float);
    
    fn_void  managed_start_  = nullptr;
    fn_float managed_update_ = nullptr;
    fn_float managed_fixed_  = nullptr;
    
    bool load_hostfxr();
    bool get_managed_entry_points();
};
```

关键实现要点:
1. `hostfxr_initialize_for_runtime_config` — 初始化 .NET 运行时
2. `hostfxr_get_runtime_delegate` — 获取 `load_assembly_and_get_function_pointer`
3. 加载 `DSEngine.Runtime.dll` → 获取 `ScriptRegistry.Initialize` 等入口
4. C# 侧用 `[UnmanagedCallersOnly]` 导出回调

#### 1.4 C# 侧反向调用入口

```csharp
// GameScripts/DSEngine.Runtime/Callbacks.cs
using System.Runtime.InteropServices;

namespace DSEngine;

public static class Callbacks {
    [UnmanagedCallersOnly]
    public static void Initialize() {
        ScriptRegistry.DiscoverAndInstantiate();
    }
    
    [UnmanagedCallersOnly]
    public static void Update(float dt) {
        ScriptRegistry.InvokeUpdate(dt);
    }
    
    [UnmanagedCallersOnly]
    public static void FixedUpdate(float dt) {
        ScriptRegistry.InvokeFixedUpdate(dt);
    }
}
```

#### 1.5 CMake 集成

```cmake
option(DSE_CSHARP "Enable C# scripting via .NET 8" OFF)

if(DSE_CSHARP)
    # 确保 dse_engine 导出为 SHARED (C# P/Invoke 需要)
    if(NOT BUILD_SHARED_LIBS)
        message(WARNING "DSE_CSHARP requires BUILD_SHARED_LIBS=ON")
    endif()
    
    target_compile_definitions(dse_engine PRIVATE DSE_CSHARP=1)
    target_sources(dse_engine PRIVATE
        engine/scripting/csharp/csharp_host.cpp
        engine/scripting/csharp/csharp_callbacks.cpp
    )
    
    # Windows: hostfxr 在 .NET SDK 安装目录
    if(WIN32)
        # 运行时自动搜索 hostfxr.dll (nethost.lib 提供)
        find_package(nethost QUIET)
        if(nethost_FOUND)
            target_link_libraries(dse_engine PRIVATE nethost::nethost)
        else()
            # 回退: 从 .NET SDK 路径直接链接
            target_link_libraries(dse_engine PRIVATE nethost)
        endif()
    endif()
    
    # 构建 C# 项目
    add_custom_target(dse_csharp_build ALL
        COMMAND dotnet build "${CMAKE_SOURCE_DIR}/GameScripts/DSEngine.sln"
                -c $<IF:$<CONFIG:Debug>,Debug,Release>
                -o "${CMAKE_RUNTIME_OUTPUT_DIRECTORY}/managed/"
        COMMENT "Building C# scripts → managed/*.dll"
    )
    add_dependencies(dse_csharp_build dse_engine)
endif()
```

### Phase 2: 热重载 + 事件系统 (1 周)

#### 2.1 AssemblyLoadContext 热重载

```csharp
// C# 侧
public static class HotReload {
    private static AssemblyLoadContext? _alc;
    
    public static void Load(string dllPath) {
        _alc = new AssemblyLoadContext("GameScripts", isCollectible: true);
        var asm = _alc.LoadFromAssemblyPath(dllPath);
        ScriptRegistry.ScanAssembly(asm);
    }
    
    public static void Unload() {
        ScriptRegistry.DestroyAll();
        _alc?.Unload();
        _alc = null;
        GC.Collect();
        GC.WaitForPendingFinalizers();
    }
}
```

C++ 侧 FileWatcher 检测 `GameScripts/` 变更 → `dotnet build` → 调用 `HotReload.Unload()` + `HotReload.Load(newDll)`。

#### 2.2 事件/回调系统 (C → C#)

```csharp
// 碰撞回调注册
public static class PhysicsCallbacks {
    [UnmanagedCallersOnly]
    public static void OnCollision(uint entityA, uint entityB, float nx, float ny, float nz) {
        ScriptRegistry.DispatchCollision(new Entity(entityA), new Entity(entityB), new Vector3(nx, ny, nz));
    }
}
```

C++ 侧在碰撞检测后通过已缓存的函数指针调用 `OnCollision`。

## 6. 风险与缓解

| 风险 | 缓解 |
|---|---|
| .NET SDK 安装要求 | 开发时需 SDK; 发布时 self-contained 或 NativeAOT |
| P/Invoke marshal 开销 (string) | 高频路径用 `Span<byte>` + stackalloc; 非关键路径可容忍 |
| 签名不匹配 → runtime crash | CI 加 `codegen --dry-run && git diff --exit-code`; .NET source generator 编译期校验 |
| 热重载中状态丢失 | DseScript 序列化 public 字段 (JSON) → Unload → Load → 反序列化恢复 |
| AssemblyLoadContext 内存泄漏 | Weak reference + 定期 GC; 保证无 static 强引用游戏 DLL 类型 |
| dse_engine 必须是 SHARED | CMake 层 `BUILD_SHARED_LIBS=ON` when `DSE_CSHARP=ON` |

## 7. 与现有 Lua 脚本共存

```
引擎主循环:
  1. LuaHost::update(dt)      — 现有 Lua 脚本照常
  2. CSharpHost::invoke_update(dt) — C# 脚本 (新增, 可选)
```

- 共享同一个 ECS World (通过 C ABI)
- Lua 和 C# 可操作同一个 Entity (通过 entity ID)
- 不冲突, 可渐进迁移

## 10. 总结

**核心设计决策:**
- 运行时: .NET 8 CoreCLR (hostfxr) — 一步到位，无迁移债务
- 绑定: `[LibraryImport]` P/Invoke — 零注册代码，编译期校验
- 反向调用: `[UnmanagedCallersOnly]` + 函数指针 — 最高性能
- 热重载: `AssemblyLoadContext` (collectible) — .NET 原生支持
- 与 Lua 共存: 同一 ECS World，通过 C ABI 共享，可渐进迁移

**无技术债设计原则:**
1. C ABI 是唯一桥接层 → 任何未来语言接入零重构
2. Codegen 单一数据源 → 三端一致性自动保证
3. Entity ID 间接访问 → 热重载安全
4. 模块化可选 (`DSE_CSHARP` CMake option) → 不影响现有构建
