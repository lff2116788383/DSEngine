## 三、Android 跨平台评估

### 3.1 依赖的平台跳转

| 子系统 | 当前平台依赖 | 移植难度 | 预估工作量 |
|:-------|:-----------|:--------:|:---------:|
| 窗口/输入 | GLFW Win32 | 中 | 0.6 人月 |
| 渲染后端 | OpenGL desktop + VK + D3D11 | 大（GLES 适配） | 1.5 人月 |
| 文件系统 | std::filesystem + Windows API | 中 | 0.3 人月 |
| 构建系统 | MSVC x64 | 大 | 0.5 人月 |
| 性能适配 | 移动端降级 | 中 | 1.0 人月 |
| **走 Vulkan Only 路线** | **已有 Vulkan 后端** | **较小** | **~2.5 人月** |
| **走全路线（GLES + Vulkan）** | | **较大** | **~4-6 人月** |

### 3.2 推荐路线

**推荐走 Vulkan Only 路线**：
- 利用已有的 Vulkan 后端
- Android 7.0+ 强制支持 Vulkan 1.0（2026 年覆盖 95%+ 设备）
- 不需要写 GLES 后端和移植 31 个 shader
- 工作量约 2.5 人月

## 四、商业引擎市场分析

### 4.1 Unity 中国区现状

| 事实 | 说明 |
|:-----|:------|
| Unity 国际版在中国 | 仍可用，未被封锁 |
| 团结引擎 | Unity China 独立分支，OpenHarmony/微信小游戏优化 |
| 社区情绪 | Unity 2023 Runtime Fee 政策导致信任危机 |
| 竞争窗口 | Unity 的信任危机 + 团结引擎不等于免费的利好 |

### 4.2 UE5 配置门槛

| 硬件 | 流畅跑 | 能跑 |
|:-----|:------|:-----|
| GPU | RTX 3060+ | GTX 1060 勉强 |
| RAM | 32GB+ | 16GB 卡顿 |
| 编译时间 | 30 分钟+ | 1 小时+ |

**独立开发者大量使用 GTX 1650/RTX 3050 笔记本，UE5 在这种配置上几乎不可用。**

### 4.3 市场缺口

```
    高配置 ┼────────────────────── UE5
          │
    中配置 ┼──────── DSE 目标生态位 ──── Unity
          │            ↑
    低配置 ┼────────────────────── Godot 4
          │
          └─────────────────────────→ 编辑器成熟度
              低 ←                 → 高
```

DSE 填补的是"渲染质量高但轻量"的 PC 引擎 niche。

## 七、关键结论

### 7.1 DSE 的真实价值

1. **渲染技术在国内开源引擎中独一档**：PBR + TAA + SSR + DDGI + NPR 9 种，无人能及
2. **开发效率极高**：2 个月 7.5 万行，在自研引擎项目中罕见
3. **代码可读性**：7.5 万行，一个人两周能读完——UE5/Unity/Godot 都做不到
4. **NPR 是最大差异化卖点**：9 种开箱即用的风格化材质，国内市场独一无二

### 7.2 DSE 的致命短板

1. **编辑器是半成品**：功能存在但从未被外部用户使用过
2. **仅 Windows**：无法满足跨平台需求
3. **0 用户基础**：无社区、无贡献者、无真实游戏产出
4. **无内容运营**：技术实力与知名度完全不匹配

### 7.3 最现实的路线

> **不做商业引擎，做"内容引擎"+ 持续开发。通过 B 站教程 + GitHub 开源建立知名度，用时间补齐编辑器与跨平台短板，深耕"轻量级 PC 独立游戏引擎"这个 niche。**

**关键时间线**：
- 现在 → 6 个月：B 站内容运营 + 编辑器打磨
- 6 → 12 个月：Android 跨平台（Vulkan Only）
- 12 → 18 个月：社区建设 + 第一款真实游戏产出

**接受 DSE 是一个 niche engine 的现实。** 目标不是打败 UE5/Unity/Godot，而是在"可读性强、渲染好的轻量引擎"这个 niche 里做到最好。
