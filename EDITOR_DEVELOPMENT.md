# DSEngine Editor - 商业版后续开发文档

## 1. 架构与定位
**DSEngine Editor** (天工开物) 是大衍引擎的核心生产力工具，采用类似于 Godot 的跨进程架构设计。
前端基于 `React` + `TypeScript` + `Electron` 构建，采用极具特色的“国产修仙复古风格” UI；后端通过 `Node-API (C++)` 桥接底层的 DSEngine 渲染与物理核心。
后续开发需在此基础上，将当前的 Mock 与基础交互打磨为具备完整项目生命周期管理能力的商业级编辑器。

---

## 2. 核心功能开发计划

### 2.1 完善场景与实体交互 (Scene & Entity Interaction)
- **目标**: 提供所见即所得 (WYSIWYG) 的高效场景编辑体验。
- **功能点**:
  - **Gizmo 交互实体化**：完善前端与 C++ 桥接的 Transform 同步，实现鼠标拖拽 Gizmo（挪移、周转、伸缩）时的平滑更新与吸附 (Snapping)。
  - **框选与多选**：支持在视口中拉框多选灵体 (Entities)，并在 Inspector 中进行属性批量修改。
  - **撤销/重做 (Undo/Redo) 系统**：基于命令模式 (Command Pattern) 记录所有场景修改（位移、创建、删除、修改材质），确保数据安全。

### 2.2 资产与资源流转 (Asset Management & Pipeline)
- **目标**: 支撑大型商业游戏的资源库管理，防止资源丢失。
- **功能点**:
  - **Meta 文件系统**：为每个导入的资源（纹理、模型、音频）生成全局唯一 UUID，编辑器内部引用基于 UUID 而非文件路径。
  - **资源面板 (Asset Browser)**：在左侧或底部增加可视化的资源管理面板，支持目录树、缩略图预览、拖拽导入 (Drag & Drop)。
  - **预制体 (Prefab) 机制**：允许将配置好的 Entity 树保存为模板（法宝/阵眼），并在场景中多次实例化。

### 2.3 专业垂直编辑器 (Specialized Editors)
- **目标**: 针对不同维度的开发提供专用工具面板。
- **功能点**:
  - **动画与时间线 (Timeline Editor)**：可视化关键帧打点，支持曲线插值调整，用于制作技能特效和过场动画。
  - **着色器连线 (Visual Shader Graph)**：提供节点式材质编辑工具，降低渲染技术门槛。
  - **粒子发射器 (Particle Editor)**：实时预览粒子效果（如火焰、剑气），调整生命周期和发射速率。

### 2.4 运行与调试 (Play In Editor & Debugging)
- **目标**: 缩短“修改-验证”的开发循环。
- **功能点**:
  - **状态隔离**：在点击“开启阵法(Play)”时，克隆当前场景状态；停止时自动恢复，防止运行时的变量修改污染静态工程。
  - **实时热重载 (Hot Reloading)**：支持 C++/TS 脚本代码的保存即生效，无需重启编辑器进程。
  - **物理与碰撞可视化**：在视口中叠加绘制碰撞盒 (AABB/OBB)、射线检测路径，辅助战斗逻辑调试。

### 2.5 跨端打包与发布 (Build & Export Pipeline)
- **目标**: 一键“飞升”，输出商业级可执行文件。
- **功能点**:
  - **构建配置面板**：提供目标平台（Windows, macOS, Linux, WASM, 移动端）、图标、屏幕分辨率、代码混淆等详细设置。
  - **资产剔除与压缩**：在打包过程中，自动剔除未引用的资源，并对纹理/音频进行目标平台的格式压缩。

## 3. 前端 UI 演进路线

**当前 UI 已经全面实现了“国产修仙复古风格”的商业化基础框架，包含以下完整视图：**
1. **法阵层级 (Hierarchy)**：左侧面板实现了实体列表树、凝聚新灵体 (创建)、剔除 (删除) 操作及选中高亮。
2. **核心视界 (Viewport)**：中央集成了渲染画布，并包含工具栏（点选、挪移、周转、伸缩）。
3. **推演控制与日志 (Toolbar & Console)**：顶部拥有开启/暂停阵法按钮；底部面板包含推演日志和详尽的神念探查（帧率、DrawCalls、延迟监控）。
4. **灵力波动 (Inspector)**：右侧面板完整展示了实体的遁法位置 (Transform)、幻象显化 (SpriteRenderer)，并实现了灵纹 (Texture) 与法诀 (Material) 的下拉指派 UI。
5. **多界域飞升 (Build)**：右侧预留了项目打包导出的跨平台选择界面。

> **下一步重点：** 前端 UI 视觉层和基础状态（React State）已完备，后续开发将完全聚焦于 **"补齐 `window.electronAPI` 桥接函数的 C++ 侧真实逻辑实现"**，以及 **"完善复杂交互组件（如基于 Canvas/WebGL 的高级 Gizmo 绘制、节点连线编辑器）"**。

## 4. 底层 C++ 桥接需求 (Node-API 规范建议)

为支撑编辑器的高级功能，`dsengine_bridge.cpp` 需补充或完善以下接口：
- `pickEntities(x, y, w, h)` - 支持视口区域的框选逻辑。
- `importAsset(filePath, options)` - 带有压缩/转换配置的资源导入通道。
- `executeCommand(cmdData)` - 接入撤销/重做系统的指令执行接口。
- `setPlayMode(bool)` - 触发底层引擎的场景状态快照与恢复。

## 5. 性能与体验考量
- **零拷贝渲染优化**：确保 `getFrameBuffer` 在跨进程传输时采用 SharedMemory 等高效机制，保证视口 60FPS 以上的流畅度。
- **React 渲染隔离**：对 Inspector 等频繁更新的面板使用 `React.memo` 和虚拟列表，防止因场景节点过多导致整个编辑器卡顿。
- **布局持久化**：记录用户调整的面板宽度和标签页状态，下次打开时自动恢复“工作区”。
