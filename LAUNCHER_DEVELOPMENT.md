# DSEngine Nexus Launcher - 商业版后续开发文档

## 1. 架构与定位
**Nexus Launcher** 作为 DSEngine 的核心入口，不仅负责引擎版本的管理与项目的启动，更是开发者生态、资源商城和技术社区的枢纽。
前端采用 `React` + `TypeScript` + `Electron` 构建，后续开发需在此基础上进行深度商业化改造。

---

## 2. 核心功能开发计划

### 2.1 用户账户与鉴权系统 (Identity & Auth)
- **目标**: 接入统一的用户中心，实现 License 管理。
- **功能点**:
  - OAuth2.0 / JWT 登录（支持 GitHub, Google, 微信等快捷登录）。
  - 订阅状态校验：启动高级版引擎时校验用户云端 License。
  - 离线模式授权 (Offline Token)：支持在无网络状态下保持 7-14 天的离线开发授权。

### 2.2 增量更新与分发网络 (Patching & CDN)
- **目标**: 告别全量下载，实现高效、省带宽的引擎更新。
- **功能点**:
  - **Chunk-based 增量更新**：比对本地与云端的清单文件（Manifest），仅下载修改过的数据块。
  - **CDN 节点加速**：针对全球用户，配置多区域的 CDN 分发（如 AWS CloudFront, 阿里云 DCDN）。
  - **后台静默下载**：Launcher 在后台低占用运行，并在检测到新版本时自动预下载。

### 2.3 高级项目管理 (Advanced Project Management)
- **目标**: 提供企业级工程管理体验。
- **功能点**:
  - **云端同步与备份**：项目配置及轻量资源自动同步至云端，实现多端无缝切换。
  - **模板引擎**：新建项目时，提供从远端拉取的最新官方模板（2D, 3D, VR, 移动端优化模板等）。
  - **项目版本迁移**：提供项目向上兼容的自动升级脚本和依赖修复工具。

### 2.4 插件与资产商城 (Marketplace)
- **目标**: 建立 DSEngine 的商业生态闭环。
- **功能点**:
  - 资产商店前端（模型、音效、脚本、插件）。
  - 支付系统集成（Stripe, 微信支付, 支付宝）。
  - 资产一键导入：在 Launcher 中购买后，直接将资产包注入到选定的本地项目中。

### 2.5 遥测与崩溃分析 (Telemetry & Crash Analytics)
- **目标**: 收集用户使用习惯与崩溃日志，指导引擎迭代。
- **功能点**:
  - 集成 Sentry 或类似 APM 工具，收集 Launcher 自身的异常。
  - 捕获引擎进程（Editor/Runtime）的 Crash Dump，并提供用户友好的上传弹窗。

## 3. 前端 UI 演进路线

**当前 UI 已经全面实现了深色科幻风格的商业化基础框架（Mock 阶段），包含以下完整视图：**
1. **身份认证 (Auth)**：左侧边栏集成了用户头像、License 状态标识。
2. **下载队列 (Download Manager)**：顶部状态栏提供了下载队列与通知中心入口，引擎版本页实现了基于进度条的下载状态 Mock。
3. **资源商城 (Store)**：以网格卡片形式展示了官方与第三方的资产包、插件。
4. **学习中心 (Learn)**：提供了官方文档和视频教程的入口视图。
5. **高级项目管理**：项目卡片已添加“云端同步”状态标识及右键菜单入口（...）。

> **下一步重点：** 前端纯视觉层已完备，后续开发将完全聚焦于 **"业务逻辑与后端接口的真实对接"**。

## 4. 后端接口需求 (API 规范建议)

为支撑上述功能，后端团队需配合提供以下 RESTful/GraphQL 接口：
- \`GET /api/v1/engines/releases\` - 获取引擎版本列表与下载链接
- \`POST /api/v1/auth/login\` - 用户登录与 Token 获取
- \`GET /api/v1/marketplace/featured\` - 获取商城推荐位内容
- \`POST /api/v1/telemetry/crash\` - 上传崩溃日志与 Dump 文件

## 5. 安全与性能考量
- **防逆向与防篡改**：使用 Electron 代码混淆（如 bytenode）保护 \`main.js\` 和 \`preload.js\`，防止破解 License 校验逻辑。
- **IPC 通信安全**：严格限制 \`preload.js\` 暴露给 Renderer 进程的 API，避免 Node.js 原生权限滥用（防范 XSS 导致 RCE）。
- **内存控制**：Launcher 长时间驻留后台时，需释放 React 视图内存或通过 Electron 的 \`tray\` 模式降低系统占用。
