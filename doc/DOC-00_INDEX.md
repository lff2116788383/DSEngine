# DOC-00 文档索引

本文档定义当前 `doc/` 目录的主线文档集合，并给出推荐阅读顺序与状态词约定。

## 1. 主线文档

当前建议重点维护以下 6 份主线文档：

- `DOC-00_INDEX.md`：文档总索引与状态词约定
- `DOC-01_ARCHITECTURE.md`：当前代码对应的真实架构说明
- `DOC-02_BUILD_AND_RUN.md`：构建、运行与基础环境要求
- `DOC-03_EDITOR.md`：当前 `editor_cpp` 编辑器说明
- `DOC-04_TESTING.md`：测试入口、CTest 组织与回归范围
- `DOC-07_ROADMAP.md`：后续演进路线与优先级

## 2. 专题文档

以下文档保留为按需阅读的专题文档：

- `DOC-05_UI_LOCALIZATION.md`：2D UI 与国际化能力说明
- `DOC-06_PARTICLE_PROFILER.md`：粒子系统与 Profiler 能力说明

## 3. 不再推荐作为主线文档的内容

- 单独的 3D 路线图文档不再作为主线维护重点
- 历史方案、参考资料、开放式概念稿不再混入当前主线文档集合
- 新文档如果不能直接对应当前代码或当前迭代目标，不建议进入主线集合

## 5. 当前主线摘要

DSEngine 当前定位为：

- 面向个人独立开发者与中小型工作室
- 以 2D Runtime 为当前稳定主线
- 支持 Lua / C++ 双业务宿主
- 提供原生 C++ 编辑器 `apps/editor_cpp/`
- 提供测试入口与基础工程化能力
- 3D 已接入，但不是当前默认稳定主线

当前迭代重点是：

- 巩固 2D Runtime 主链
- 强化 Editor 可用性
- 收口资源链路与测试门禁
- 在 2D 闭环稳定后推进 3D MVP

## 6. 状态词约定

后续文档统一使用以下状态词：

- **已稳定**：代码中已有稳定入口，且属于当前默认主线或稳定验证范围
- **已实现**：代码中已有明确实现，但未必属于默认稳定主线
- **已接入**：代码中存在主流程、组件或入口，但仍需收口或补充验证
- **规划中**：尚未形成稳定代码闭环
- **不再保留**：已删除或不再维护的旧方案 / 旧文档

## 7. 文档维护原则

- 文档必须优先服从代码现状
- 文档不把远期目标写成已完成事实
- 文档不把“已接入”写成“已稳定”
- README 只保留对外定位、边界、快速开始和核心索引
- 更细的模块能力说明放入专题文档，不与主线架构文档重复
