# DSEngine 现代全栈引擎架构

> 基于**2D引擎轻量化架构**深度兼容**现代化3D引擎核心能力**，采用统一ECS数据驱动、分层解耦、跨平台抽象设计，实现2D/3D混合渲染、无缝切换、资源复用，兼顾2D项目的高性能与3D项目的复杂渲染需求，打造一套可裁剪、可扩展的全栈游戏引擎架构。

## 一、统一核心设计原则

- **数据与行为分离**：全引擎采用 ECS (Entity-Component-System) 架构，2D/3D 组件共用实体 ID，支持同场景 2D/3D 对象混合交互。
- **模块化可插拔**：3D 模块为可选扩展，纯 2D 项目可禁用 3D 相关子系统，避免性能损耗。
- **统一底层抽象**：平台层、核心库、资源管理器、事件总线完全共用，屏蔽 2D/3D 底层差异。
- **性能分级适配**：2D 场景走轻量化批处理管线，3D 场景启用 PBR、延迟渲染等复杂管线。
- **跨平台统一渲染**：底层抽象对接 Vulkan/Metal/WebGPU 等现代图形 API，2D/3D 渲染共用后端。
- **编辑器一体化**：同一编辑器支持 2D/3D 场景编辑、资源预览、调试剖析。

## 二、全栈引擎分层架构模型

```Plain Text
┌─────────────────────────────────────────────┐
│        Game Logic / Scripting / Blueprint   │  游戏逻辑层（C++/Lua通用脚本）
├─────────────────────────────────────────────┤
│        Scene / ECS World / Entity Manager   │  场景与ECS管理层（2D/3D实体统一管理）
├─────────────────────────────────────────────┤
│  ┌─────────┐  ┌─────────┐  ┌─────────────┐  │
│  │ Render  │  │ Physics │  │  Aux System │  │  功能子系统层（2D/3D分路适配）
│  │2D+3D统一│  │2D+3D兼容│  │音频/动画/UI │  │
│  └─────────┘  └─────────┘  └─────────────┘  │
├─────────────────────────────────────────────┤
│        Resource / Asset Manager             │  资源管理层（2D/3D资源统一加载、复用）
├─────────────────────────────────────────────┤
│  Core (Math/Thread/Job/Event/IO/Memory)     │  核心库层（2D/3D完全共用）
├─────────────────────────────────────────────┤
│        Platform Abstraction Layer (PAL)     │  平台抽象层（窗口/输入/硬件/图形API抽象）
└─────────────────────────────────────────────┘
```

## 三、核心模块深度解析

### 3.1 平台抽象层 (PAL)
屏蔽跨平台差异，保留原有 2D 引擎跨平台能力，扩展 3D 所需的图形 API 特性（如深度测试、模板缓冲），窗口、输入、文件接口完全共用。

### 3.2 核心库层 (Core)
在原有 2D 数学库、并发基础上，扩展 3D 工具。
- **数学库**：Vec2/Vec3/Vec4, Mat4, Quaternion, AABB/OBB，SIMD 指令优化。
- **并发与内存**：Job System、对象池复用，新增 3D 几何体内存池。
- **事件总线**：低耦合消息机制，跨模块通信。

### 3.3 资源管理层 (Asset System)
保留 2D 资源加载与热重载逻辑，扩展 3D 导入管线：
- **通用资源**：纹理、字体、音频、着色器完全共用。
- **3D扩展资源**：FBX/GLTF 模型、骨骼动画、光照贴图异步加载。
- **流式加载**：大尺寸资源后台异步加载。

### 3.4 ECS 核心架构
- **Entity (实体)**：唯一 uint64 ID，支持动态挂载 2D/3D 组件。
- **Component (组件)**：纯数据。包含 2D (Transform2D, Sprite2D, Collider2D) 与 3D (Transform3D, MeshRenderer, Material, Collider3D) 组件，以及通用组件 (Name, AudioSource)。
- **System (系统)**：2D 系统与 3D 系统并行执行，互不干扰，支持跨域数据交互。

### 3.5 渲染系统 (核心兼容设计)
采用**统一渲染后端 + 分路渲染前端**。
- **统一后端**：管理管线、帧缓冲、纹理、GPU 缓冲，减少状态切换。
- **2D 前端**：精灵批处理、纹理图集、MSDF字体、UI批处理。极低 Draw Call。
- **3D 前端**：PBR 物理光照、延迟渲染、实时阴影、后处理(Bloom, AO)。
- **混合渲染**：通过深度缓冲与模板缓冲实现层级叠加（如 3D 场景中的 2D HUD）。

### 3.6 物理系统
- **2D 物理**：Box2D / Chipmunk2D 集成。
- **3D 物理**：插件化集成 Jolt / PhysX，独立物理线程。

## 四、技术栈概览
- **核心语言**：C++17/20（引擎底层）、Lua / TS（逻辑脚本）
- **图形API**：抽象兼容 Vulkan、Metal、WebGPU、OpenGL
- **物理引擎**：Box2D (2D) + Jolt/PhysX (3D)
- **ECS框架**：EnTT
- **构建工具**：CMake

## 五、兼容与扩展优势
- 原有 2D 引擎项目无需重构即可迁移。
- 按需启用 3D 模块，纯 2D 项目剔除 3D 代码，无性能损耗。
- 分层解耦设计使得引擎可以在轻量化与工业化之间自由切换。
