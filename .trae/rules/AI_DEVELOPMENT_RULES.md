# DSEngine AI 开发规则

## Description
DSEngine 项目的核心 AI 开发规则，规定了 C++, Lua, React 混合架构下的工作流、技术栈约束和防幻觉底线。

## Globs
*.*

## Rule Governance
- **规则集版本**: `v1.0.0`
- **生效日期**: `2026-03-26`
- **冲突优先级**:
  1. `AI_DEVELOPMENT_RULES.md`（总纲）
  2. `AI_TESTING_RULES.md`（测试约束）
  3. `AI_DOC_STANDARDS.md`（注释与文档表达）
- **裁决原则**: 出现冲突时按优先级执行；低优先级规则不得否定高优先级硬约束。

## Content
### 1. 核心原则
- **谋定而后动**: 修改前必须使用工具全面检索代码库，禁止凭空猜测API。
- **最小化变更**: 优先修改现有文件，避免不必要的新建和重构。
- **一致性**: 严格遵循现有命名规范、缩进和设计模式(如C++、Lua、React的不同惯用法)。
- **闭环验证**: 代码变更后必须通过编译、测试或Shell命令验证，不交付未验证代码。

### 2. 技术栈约束
- **C++ (engine/)**: 核心层。要求高性能、内存安全，遵循现代C++规范。
  - **内存管理**: 禁用裸指针(`new`/`delete`)进行所有权管理。优先使用现有的内存池和对象池，常规动态分配必须使用智能指针。
  - **错误处理**: 引擎层默认禁用异常(`-fno-exceptions`)。使用断言(`assert`或自定义宏)处理致命错误，常规错误返回错误码或`std::optional`。
- **Lua (script/ & samples/)**: 脚本层。遵循表驱动编程，注意C++/Lua交互成本。
- **React/TS (apps/)**: 工具层。遵循现代前端规范，使用Hooks和严格类型检查。
- **CMake**: 新增文件/库时必须同步更新`CMakeLists.txt`。

### 3. 工作流
1. **分析**: 拆解任务，检索上下文，有歧义主动询问(AskUserQuestion)或合理假设。
2. **规划**: 复杂任务(>3步)立即使用TodoWrite建立任务列表。
3. **执行**: 
   - 修改前必须 Read 文件，SearchReplace 保证上下文唯一性。
   - **防乱码与破坏**: 写入文件时注意保留原有的 BOM 编码头，严禁使用低级正则脚本进行无脑的批量代码/注释注入，所有修改必须基于准确的语义解析。
4. **交付**: 验证后输出，回复必须带代码引用(Markdown链接)。

### 4. 防幻觉
- **禁伪造API/库**: 必须在代码库或`depends/`/`package.json`中确认存在。
  - **第三方库**: 严禁未经用户明确授权擅自引入新的第三方库（如通过 vcpkg/git submodule 等）。
  - **跨语言绑定**: 编写 Lua 脚本时，严禁臆造未经 C++ 导出的 API。必须先检索 C++ 绑定代码确认该 API 已真实导出给 Lua。
- **禁高危命令**: 构建/清理命令局限于项目工作区。
- **禁虚假完成**: 遇阻时Todo保持`in_progress`并诚实说明，绝不标记完成。

### 5. 架构保护
- 修改核心系统(`ecs/`, `render/`, `core/`)极其谨慎。
- 遵循现有事件总线和作业系统，禁随意引入新并发/事件机制。
- 架构变动必须同步更新对应Markdown文档。
