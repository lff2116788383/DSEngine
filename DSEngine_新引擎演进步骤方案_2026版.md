# DSEngine 新引擎演进步骤方案（2026版）

> 目标：基于本地已有架构与实现（ECS、RHI、C++/Lua双宿主等），重新规划AI辅助演进路线，最终实现全栈商业化引擎。
> 架构拆分原则：
> - **Launcher（轻量独立启动器）**：极小体积，仅负责登录、选择引擎版本、选择/创建项目、引擎热更新。
> - **Editor（编辑器本体）**：仅负责场景编排、资源编辑、游戏运行测试，绝对不负责登录、版本管理与热更新。

---

## 1. DSEngine 第一阶段 2D MVP版本
**目标**：在现有 Phase1 主干上形成“可玩、可测、可回归”的现代 2D 引擎最小完整体，跑通 Sprite + Physics + UI + Audio + Tilemap + Animation 的统一运行链路，保持 C++/Lua 双宿主一致行为，不引入 3D 越界实现。

### 1.1 引擎库 (Engine Core)
- **底层基建**：基于 Job System 的跨平台多线程任务调度，完善的 RHI 接口抽象（CommandBuffer, Pipeline, Buffer, Texture），使用全维度数学库（Mat4）支撑 2D 变换。
- **ECS 架构**：深度集成 EnTT，确保组件数据与系统逻辑完全分离，杜绝 GameObject 模式。
- **全链路 2D 系统集成**：实现并跑通 `Sprite`（极速动态批处理）、`Physics`（Box2D 刚体碰撞）、`UI`（基础控件渲染）、`Audio`（音频播放）、`Tilemap`（瓦片地图渲染）与 `Animation`（动画控制）的统一 ECS 运行循环。
- **资源管理**：实现基础资源（Texture、Shader、AudioClip）的加载与引用计数，支持异步回调。

### 1.2 Lua 脚本 (Lua Scripting)
- **双宿主一致性**：C++ 与 Lua 业务层完全解耦，统一复用 Engine App 生命周期，确保两者行为与运行链路一致。
- **全链路接口绑定**：将上述 Sprite、Physics、UI、Audio、Tilemap、Animation 的底层核心组件与系统接口完整绑定至 Lua。
- **综合业务验证**：在 `examples/lua` 中构建“可玩、可测、可回归”的 2D 综合测试场景（涵盖精灵物理碰撞、UI交互响应、音效播放、Tilemap渲染与动画播放）。

### 1.3 轻量独立启动器 (Launcher)
- **极简 UI**：使用轻量级原生框架实现最小 MVP，保证极小体积。
- **项目管理**：支持扫描本地目录，选择已有的项目并启动对应的 Editor。
- **无热更模拟**：此时无需真实的热更新下载，只需跑通引擎版本列表的选择交互逻辑。

### 1.4 编辑器 (Editor)
- **桥接渲染**：基于现有前端框架打通 C++ 引擎的渲染视口（共享内存/帧缓冲传输）。
- **基础场景树**：实现左侧 Scene 实体列表，支持创建/删除实体（涵盖各 2D 组件实体）。
- **属性面板**：实现右侧 Inspector，支持修改 Transform 及 Sprite、Physics、UI 等组件的基础属性。
- **游戏运行控制**：在编辑器内加载与运行综合 MVP 场景，实现视口鼠标拾取（Picking）。

## 2. DSEngine 第一阶段 2D 商业版本
**目标**：在 2D MVP 基础上补全商业化必需的生产力工具与工程化系统，实现完整的 2D 游戏制作工作流、自动化回归与性能达标。

### 2.1 引擎库 (Engine Core)
- **高级 2D 系统与管线（规划新增）**：引入 Spine 2D 骨骼动画支持，完善基于数据驱动的复杂 UI 布局与事件流，实现高级音频混音与总线控制。
- **资源进阶（规划新增）**：实现 Prefab 系统（序列化/反序列化）、资源打包与加密（Bundle），GPU 资源生命周期严格管理（防泄漏）。
- **多实例与解耦**：完全去除单例（AssetManager/Runtime Context 注入），支持多实例并行测试与运行。
- **SDK 化输出**：完善 `install/export` 流程，规范头文件、lib、dll 输出，提供标准 C++ 与 Lua 最小接入模板与 CI/CD 构建脚本。

### 2.2 Lua 脚本 (Lua Scripting)
- **高级业务绑定（规划新增）**：绑定 Spine 动画控制、高级 UI 布局事件、音效总线、资源打包加载（Bundle/Prefab）等进阶接口。
- **热重载支持**：实现 Lua 脚本在运行时的无缝热重载机制，提升业务开发迭代效率。
- **性能与调试**：提供 Lua 内存泄漏检测机制与性能 Profiler（性能剖析器）支持，确保业务脚本在商业项目中的稳定运行。
- **业务框架**：提供一套基于 Lua 的标准游戏生命周期框架（State Machine、UI 管理器等）。

### 2.3 轻量独立启动器 (Launcher)
- **用户登录**：接入账号体系，实现开发者登录验证功能。
- **引擎版本控制**：拉取云端引擎版本配置，实现引擎版本的下载与本地管理。
- **引擎热更新**：实现增量下载（基于 Hash 对比）与引擎核心库（DLL/脚本环境）的热更新。
- **项目创建**：提供标准项目模板（基于 Lua 或 C++），一键生成工程目录结构并完成初始化。

### 2.4 编辑器 (Editor)
- **资源与资产管理**：完善左侧 FileSystem，支持图片资源的拖拽导入、材质参数化编辑（Material Variants）。
- **实时热更预览**：材质、着色器修改后，视口实时回放与热更新。
- **打包流水线**：集成一键导出（Export Project），调用后端构建脚本发布目标平台包。
- **性能剖析器**：在编辑器中集成 DrawCall、Batch、实体数、内存等核心性能指标的可视化图表。

## 3. DSEngine 第二阶段 3D MVP版本
**目标**：在不破坏 2D 架构的前提下，通过系统与组件的横向扩展验证 3D 渲染与物理的核心可行性，保持 2D 作为独立模块的纯净度。

### 3.1 引擎库 (Engine Core)
- **3D 核心组件与系统**：以插件/模块化形式新增 `MeshRendererComponent`、`LightComponent`（平行光/点光）、`CameraComponent`（透视投影），以及对应的 `MeshRenderSystem`。
- **3D 资源解析**：引入 Assimp/cgltf 加载 GLTF/FBX 模型资源，作为 Asset Manager 的扩展解析器。
- **基础 3D 渲染管线**：实现基础的 Forward Rendering（前向渲染），支持简单的漫反射与方向光照。
- **3D 物理验证**：引入 Jolt Physics 或 PhysX 模块，实现 `RigidBody3DComponent` 与基础的物理模拟碰撞（需与 Box2D 的 2D 物理彻底隔离）。

### 3.2 Lua 脚本 (Lua Scripting)
- **3D 接口暴露**：将 Mesh 实例化、3D 坐标变换、3D 物理施加力等基础操作开放给 Lua。
- **3D MVP 测试用例**：在 `examples/lua` 中构建 3D 基础测试场景（加载带纹理的模型并赋予 3D 物理掉落效果）。

### 3.3 轻量独立启动器 (Launcher)
- **平滑过渡**：版本列表中增加“3D 测试版”通道，不影响 2D 商业项目的热更与启动。
- **模板新增**：在创建项目时，新增“3D 基础项目”模板供开发者选择。

### 3.4 编辑器 (Editor)
- **3D 视口升级**：编辑器视口支持 3D 自由相机漫游（WASD + 鼠标右键旋转）。
- **3D 拾取与坐标轴**：实现 3D 空间内的射线拾取（Raycasting）以及位移/旋转/缩放的 3D Gizmo 工具。
- **基础材质编辑**：支持将 3D 模型纹理槽（Albedo）映射并开放给属性面板调整。

## 4. DSEngine 第二阶段 3D 商业版本
**目标**：将 3D 渲染管线推向现代 PBR 标准，完善 3D 工作流，实现 2D/3D 混合渲染，满足次世代 3D 商业游戏开发需求。

### 4.1 引擎库 (Engine Core)
- **现代 3D 渲染管线 (Render Graph)**：实现基于物理的渲染（PBR），支持 Deferred Rendering（延迟管线）。
- **高级光影**：实现级联阴影贴图（CSM）、点光源阴影、环境光遮蔽（SSAO）与全局光照（GI）雏形。
- **后处理堆栈 (Post-Processing)**：实现 Bloom、Tone Mapping、景深（DoF）、抗锯齿（TAA/FXAA）。
- **2D/3D 混合**：确保阶段一的 2D 系统可作为 HUD 或场景元素无缝渲染在 3D 场景中。
- **资源流式加载**：全面实现大尺寸纹理、网格、音频的后台流式异步加载与卸载。

### 4.2 Lua 脚本 (Lua Scripting)
- **高级 3D 控制**：绑定骨骼动画控制器、光照参数动态修改、材质属性运行时替换等接口。
- **性能优化**：对 Lua 访问密集型 3D 组件数据（如 Transform 大规模更新）提供批量 C++ 处理接口，避免跨语言开销。

### 4.3 轻量独立启动器 (Launcher)
- **模块化下载**：由于 3D 引擎体积增大，启动器支持按需下载组件包（如只下 2D 核心包、附加 3D 渲染包、附加高级物理包）。
- **项目升级诊断**：提供旧版 2D 项目升级到混合 3D 引擎的兼容性扫描与报告功能。

### 4.4 编辑器 (Editor)
- **完善的 3D 场景构建**：支持地形编辑（Terrain）、植被刷子、高级光照烘焙与探针布置。
- **复杂材质编辑器**：实现基于节点（Node-based）的可视化 Shader 编辑器。
- **动画状态机**：可视化编辑 3D 骨骼动画状态机与混合树（Blend Tree）。
- **全功能发布**：完善 3D 资源的压缩策略与分包策略，支持全平台高性能发布。
