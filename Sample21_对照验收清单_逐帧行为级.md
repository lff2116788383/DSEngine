# Sample21 对照验收清单（逐帧行为级）

## 验收范围
- 目标：对齐 `GN_SDK1e/Sample/base/Sample21.cpp` 的视觉、输入、音频、动画、计分、状态机与时序行为。
- 粒度：按“逐帧可观察行为”验收，而不是仅看功能有无。
- 判定：每项给出通过标准（可量化/可录屏对比）。

## A. 启动与页面状态机（逐帧）
- [ ] A1 首帧显示开始页背景与开始按钮，非游戏场景元素透明或隐藏。
- [ ] A2 开始页星星粒子持续发射，密度稳定，无突变。
- [ ] A3 鼠标悬停开始按钮时，按钮有放大/高亮反馈（1-3 帧内出现变化）。
- [ ] A4 点击开始按钮后，下一帧切入游戏态：开始页特效衰减、游戏层可见。
- [ ] A5 游戏结束后进入 GameOver 态，按钮可再次触发重开，重开后状态完全重置。

## B. 背景与分层渲染
- [ ] B1 背景层、草地层、叶子层、角色层、前景层、UI层顺序稳定，无穿插错层。
- [ ] B2 草地 UV 滚动与镜头上移同步，滚动方向和速度与原作一致（肉眼无反向感）。
- [ ] B3 前景 6 张图在镜头移动时保持“屏幕钉住”效果，不随世界坐标漂移。
- [ ] B4 波纹/水花在遮挡关系上应位于叶子/角色下方或上方与原作一致（需录屏确认）。
- [ ] B5 UI 数字始终在最上层，不被场景精灵覆盖。

## C. 叶子无限场景（核心）
- [ ] C1 初始叶子数量与可视密度接近原作，开局 2 秒内无空档带。
- [ ] C2 叶子横向往返移动，触边反向，速度有轻微随机差异。
- [ ] C3 镜头上移后，离开底部视野的叶子被复用到顶部，连续 60 秒无“断层”。
- [ ] C4 叶子重生时最小间距满足设计，不出现两片严重重叠导致必死局。
- [ ] C5 角色落叶瞬间叶片有“压缩/下沉感”（如有实现），恢复应平滑无抖动。

## D. 角色动作与物理时序
- [ ] D1 待机：落叶后下一帧切回待机循环。
- [ ] D2 起跳：松开按键后 1 帧内切换跳跃动作。
- [ ] D3 落水：未命中叶子时切换落水动作，动作结束后触发复活流程。
- [ ] D4 重生：复活前等待时间与原作量级一致（约 1 秒），期间不可再次起跳。
- [ ] D5 水平跟随：落地状态角色 X 向当前目标叶子平滑靠拢，无瞬移。
- [ ] D6 纵向运动：重力衰减连续，速度曲线无“台阶感”。

## E. 输入与蓄力（逐帧响应）
- [ ] E1 按下左键开始计时，持续按住不触发提前起跳。
- [ ] E2 松开左键触发起跳，按住时长越长，初速度越大（上限生效）。
- [ ] E3 蓄力时间极短时仍可小跳，不出现 0 速无动作。
- [ ] E4 蓄力极长时被速度上限钳制，峰值稳定。
- [ ] E5 右键取消蓄力（如设计对齐原作）时应立即失效，不残留旧计时。

## F. 镜头追随（2D伪相机）
- [ ] F1 镜头 Y 追随角色，采用平滑阻尼，不锁死角色中心。
- [ ] F2 镜头移动时，波纹/水花等局部特效与世界坐标一致，不出现“漂浮在屏幕”。
- [ ] F3 镜头快速上移时，UI 不受影响，世界层无明显抖动。
- [ ] F4 连续跳跃 30 秒后，镜头误差不累计（角色仍处于期望可视区）。

## G. 音频与时机
- [ ] G1 BGM 启动后循环播放，无首尾爆音。
- [ ] G2 起跳音效在“松开触发帧”播放，不早不晚。
- [ ] G3 起跳音调随速度变化：短按与长按可明显区分音高。
- [ ] G4 落地、落水、连击、失败音效触发时机与事件一致，且不互相吞音。
- [ ] G5 高频触发时（连跳/连击）音效并发受控，无明显失真或爆量叠音。

## H. 粒子与组合特效
- [ ] H1 开始页星星：持续稳定发射，切游戏态后按设计衰减或关闭。
- [ ] H2 连击粒子：连击升高时密度提升，停止连击后逐步回落。
- [ ] H3 落水时有一次性 burst，位置与角色落水点一致。
- [ ] H4 粒子寿命、透明衰减连续，无“突然消失整批”。
- [ ] H5 粒子与主渲染帧率波动时不应明显卡顿跳帧。

## I. 计分/连击/生命/最高分
- [ ] I1 成功落叶后得分增加，且与跨越叶子数/连击倍率相关。
- [ ] I2 连击窗口内连续高质量落叶触发 Combo 递增，超时后清零或衰减。
- [ ] I3 生命值在失败时递减，显示不出现负数。
- [ ] I4 最高分在本局超越时实时更新，重开后保留。
- [ ] I5 GameOver 后再次开始，本局分/连击/通关叶数归零，最高分保留。

## J. 边界与稳定性
- [ ] J1 连续快速点击开始按钮不应生成重复实体或重复音源。
- [ ] J2 长时间运行（>10 分钟）无明显内存上涨、实体数量失控。
- [ ] J3 极端操作（超长蓄力、连续失败重开）不导致状态机卡死。
- [ ] J4 分辨率变化后按钮命中、UI位置、世界坐标映射仍正确。
- [ ] J5 资源缺失时应可定位错误（日志明确），不应静默黑屏。

## K. 对照建议（怎么判定“完整复刻”）
- [ ] K1 录两段视频：原版 Sample21 与 DSE 版，同操作脚本（30-60 秒）。
- [ ] K2 按本清单逐项打勾：通过/偏差/失败。
- [ ] K3 “完整复刻”建议门槛：A~I 必须全通过，J 至少 4/5 通过。
- [ ] K4 若有偏差，优先修复顺序：输入与动作时序 > 镜头 > 计分 > 音频 > 特效观感。
